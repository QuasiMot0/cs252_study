## What Is ARENA_SIZE?

`ARENA_SIZE` is the number of bytes requested from the OS in a single `sbrk()` call. It is a parameterized constant defined in `myMalloc.h`. The tests set it individually: `ARENA_SIZE=1024` (1KB) in the provided test cases.

Rather than calling `sbrk()` for every individual `malloc()` request, the allocator requests memory in large chunks and then parcels that chunk out to satisfy individual requests.

## Lazy Allocation

New chunks are allocated **lazily** — only when a request cannot be satisfied by any block currently in the free lists:

1. Search free lists for a block large enough to satisfy the request.
2. If found: split if needed, return to user.
3. If **not found**: call `sbrk(ARENA_SIZE)` to get a new chunk.

This avoids requesting more memory from the OS than is needed.

## Handling Requests Larger Than ARENA_SIZE

If a single allocation request is larger than `ARENA_SIZE`, the allocator must make **multiple consecutive `sbrk()` calls** to obtain enough contiguous memory:

```
User requests 1000 bytes, ARENA_SIZE = 400 bytes:
  → sbrk(400) → sbrk(400) → sbrk(400)  (three calls to get 1200 total bytes)
  → Coalesce into one 1200-byte block
  → Allocate 1000 bytes from it
```

All new chunks must be of size `ARENA_SIZE`. This is how the tests expect the allocator to handle new chunks.

## Two Cases When a New Chunk Arrives

### Case 1: New chunk is adjacent to the previous chunk

When two consecutive `sbrk()` calls return contiguous memory (no intervening `sbrk()` call by another library):

- Remove the fenceposts between the two chunks.
- Coalesce the new chunk with any unallocated memory at the end of the previous chunk.
- This forms a single larger block.

### Case 2: New chunk is not adjacent to the previous chunk

Another library called `sbrk()` between our calls, creating a gap:

- The new chunk cannot be coalesced with the previous chunk.
- Insert the new chunk into the free list as a standalone block (with its own fenceposts).

## Handling sbrk() Failure

`sbrk()` can fail if the OS has no memory to give. Per the man page:

- Return `NULL` to the user.
- Set `errno = ENOMEM`.

```c
void *new_chunk = sbrk(ARENA_SIZE);
if (new_chunk == (void*)-1) {
    errno = ENOMEM;
    return NULL;
}
```

## Key Terms

| Term | Meaning |
|---|---|
| `ARENA_SIZE` | Size of each OS memory request (1KB in tests); parameterized |
| Lazy allocation | Request from OS only when free lists cannot satisfy a request |
| Adjacent chunks | Two consecutive `sbrk()` returns that are contiguous; can be coalesced |
| `ENOMEM` | Error code set when `sbrk()` fails (out of memory) |
