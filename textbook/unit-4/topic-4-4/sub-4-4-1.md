## The Allocation Algorithm

From the slides, the full allocation procedure:

1. **Round up** requested size to the next 8-byte boundary. If size==0, treat as 1 byte.
2. **Compute actual size**: `actual_size = roundup8(requested_size) + sizeof(allocated_metadata)`
3. **Find a free block** large enough in the appropriate free list (or the next non-empty list).
4. **Split or use whole block** (see below).
5. **If no block exists**: request a new chunk from the OS via `sbrk(ARENA_SIZE)`.

## To Split or Not to Split

When the found free block is larger than needed, there are two sub-cases:

### Case A: Remainder is too small

If `found_block->size - actual_size < sizeof(full_header)`:

- The remainder would be too small to function as a standalone free block (it couldn't hold `next` and `prev` pointers).
- **Do not split.** Allocate the entire block to the user — the extra bytes become internal fragmentation.

From the slides:
> "If the remainder is smaller than the size of the header plus the next/prev making the object unusable, then do not split the object and use the entire object to satisfy the allocation."

### Case B: Remainder is large enough

If `found_block->size - actual_size >= sizeof(full_header)`:

- Split the block into two: the allocated portion and the free remainder.
- The **right (higher in memory) portion** is allocated to the user.
- The **left (lower in memory) portion** is the remainder and stays in (or moves to) a free list.

**Why allocate the right half?** The lab specification requires this for **determinism** — the reference implementation always splits from the right so test outputs match exactly.

## After Splitting

The allocator must update metadata for both new blocks and for the block immediately to the right of the newly allocated block:

1. Left remainder block: update `size` to `found_block->size - actual_size`.
2. New allocated block: set `size = actual_size`, `left_size = left_remainder->size`, `allocated = 1`.
3. Block to the right of the new block: update its `left_size = actual_size`.
4. Insert the left remainder into the appropriate free list. If its size no longer falls in the original list's range, move it to the correct list.

## HW1 Example: malloc(8) into a 64-byte free block

```
Before:  [Block 1: size=64, allocated=0]

actual_size = roundup8(8) + sizeof(header) = 8 + 24 = 32

64 - 32 = 32 ≥ sizeof(full_header) → SPLIT

After:   [Block 1: size=32, allocated=0]  [Block 3: size=32, allocated=1]
          (remainder, free, left half)       (allocated to user, right half)

Block 3's left_size = 32 (size of Block 1)
Next block's left_size updated to 32 (size of Block 3)
Returns: address of Block 3 + sizeof(allocated_metadata)
```

## Key Terms

| Term | Meaning |
|---|---|
| Splitting | Dividing a free block into an allocated portion and a free remainder |
| Right half | The allocated portion (higher in memory) — required by CS252 spec for determinism |
| Minimum remainder | `sizeof(full_header)` — below this, don't split; use entire block |
| Internal fragmentation | Wasted space when entire block used but only part needed |
