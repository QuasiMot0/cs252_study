## What realloc() Does

`realloc(ptr, size)` changes the size of the previously allocated block pointed to by `ptr`. The returned pointer may or may not be the same as `ptr`.

From the C standard:
- If the new size is smaller, the existing block may be truncated.
- If the new size is larger, new storage is obtained. Data up to the old size is preserved; new bytes are uninitialized.

## Standard Implementation

The basic (non-optimized) implementation:

```c
void *realloc(void *ptr, size_t size) {
    if (ptr == NULL) return malloc(size);      // realloc(NULL, n) == malloc(n)
    if (size == 0) { free(ptr); return NULL; } // realloc(ptr, 0) == free(ptr)

    void *new_ptr = malloc(size);
    if (new_ptr == NULL) return NULL;

    // Copy old data (up to min of old and new sizes)
    size_t old_size = get_block_size(ptr) - sizeof(allocated_metadata);
    memcpy(new_ptr, ptr, min(old_size, size));

    free(ptr);
    return new_ptr;
}
```

This always allocates a new block, copies data, and frees the old block.

## Optimization: Grow In Place

The lab handout describes an extra-credit optimization:

> "Check if the block beyond the old object is free, and then extending the old object into this space."

If the right neighbor of the block being reallocated is free AND `current_block->size + right_neighbor->size >= new_actual_size`:

1. Remove the right neighbor from its free list.
2. Extend the current block to absorb the right neighbor: `current_block->size += right_neighbor->size`.
3. Update the block after the right neighbor: `left_size = current_block->size`.
4. No copy is needed — the data is already in place.
5. Return the same pointer `ptr`.

This avoids the expensive `malloc` + `memcpy` + `free` sequence.

## Special Cases

| Call | Behavior |
|---|---|
| `realloc(NULL, size)` | Equivalent to `malloc(size)` |
| `realloc(ptr, 0)` | Equivalent to `free(ptr)`; returns NULL |
| `realloc(ptr, same_size)` | Implementation-defined; may return same pointer |

## Why Free the Original on Move

If `realloc` allocates a new block (because in-place growth is not possible), the old block must be freed. Failing to free it is a **memory leak**. The original pointer should not be used after a successful `realloc` call — the original block may now be gone.

## Key Terms

| Term | Meaning |
|---|---|
| `realloc(ptr, size)` | Resize a previously allocated block; data preserved up to old size |
| Grow-in-place | Absorb adjacent free block without copying data (extra credit optimization) |
| `realloc(NULL, n)` | Equivalent to `malloc(n)` |
| `realloc(ptr, 0)` | Equivalent to `free(ptr)`, returns NULL |
| Memory leak | What happens if the old block is not freed after realloc copies to a new location |
