## Why Cast to char*?

In C, pointer arithmetic uses the **size of the pointed-to type** as the unit. If `p` is a `BoundaryTag*`, then `p + 1` advances by `sizeof(BoundaryTag)` bytes, not 1 byte.

To move by an arbitrary byte offset (such as the block's actual size stored in `size`), you must cast to `char*` first:

```c
// WRONG: advances by sizeof(BoundaryTag) * block->size bytes
BoundaryTag *right = block + block->size;

// CORRECT: advances by exactly block->size bytes
BoundaryTag *right = (BoundaryTag*)((char*)block + block->size);
```

## Key Address Computations

### Right neighbor
```c
BoundaryTag *right_neighbor = (BoundaryTag*)((char*)block + block->size);
```

### Left neighbor
```c
BoundaryTag *left_neighbor = (BoundaryTag*)((char*)block - block->left_size);
```

### Header from user pointer (what malloc returns)
The user receives a pointer to the **data** portion, just past the allocated metadata:

```c
// Converting user pointer → header:
BoundaryTag *header = (BoundaryTag*)((char*)user_ptr - sizeof(allocated_metadata));

// Converting header → user pointer:
void *user_ptr = (void*)((char*)header + sizeof(allocated_metadata));
```

Where `sizeof(allocated_metadata)` is `sizeof(size)` + `sizeof(left_size)` + `sizeof(allocated)` (the fields always present, without `next`/`prev`).

## Why Block Addresses Must Be Updated After Split

When a block is split, the right half goes to the user. The left half is the remainder. The block immediately after the right half must have its `left_size` updated to reflect the right half's new actual size.

Similarly, when blocks coalesce, the block immediately to the right of the merged block must have its `left_size` updated to the merged block's new (larger) size.

## HW1 Example: malloc(8) on a 64-byte free block

Arena starts at address `...0000`, dummy fencepost has size 16.

Initial state:
- Block 1 at `...0000`: size=64, left_size=16, allocated=0

After `malloc(8)` (allocates rightmost portion):
- Block 1 at `...0000`: size=32, left_size=16, allocated=0 (remainder, stays free)
- Block 3 at `...0032`: size=32, left_size=32, allocated=1 (new allocated block)
- Block 2 at `...0064`: left_size updated to 32

Returns to user: `...0032 + sizeof(BoundaryTag)` (pointer past the header).

## Key Terms

| Term | Meaning |
|---|---|
| `(char*)` cast | Required to do byte-level pointer arithmetic in C |
| `(char*)block + block->size` | Address of the right neighbor |
| `(char*)block - block->left_size` | Address of the left neighbor |
| `sizeof(allocated_metadata)` | Offset from block start to user data |
| `left_size` update | Required after split or coalesce to keep the right neighbor's metadata correct |
