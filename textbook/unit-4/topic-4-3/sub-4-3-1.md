## The Header Struct (BoundaryTag)

Every block in the heap — whether allocated or free — begins with a header (`BoundaryTag` in the CS252 lab code) containing metadata:

```
┌──────────────┐
│    size      │  8 bytes — actual total size of this block
├──────────────┤
│  left_size   │  8 bytes — actual size of the left neighbor block
├──────────────┤
│  allocated   │  4 bytes — 1 if allocated, 0 if free (or magic number in debug mode)
├──────────────┤
│    next *    │  8 bytes — (free blocks only) pointer to next block in free list
├──────────────┤
│    prev *    │  8 bytes — (free blocks only) pointer to prev block in free list
├──────────────┤
│  user data   │  ← malloc returns a pointer here
└──────────────┘
```

**Allocated metadata** (always present): `size`, `left_size`, `allocated`

**Unallocated metadata** (additionally present when free): `next`, `prev`

## Optimizing Metadata Size

When a block is **allocated**, `next` and `prev` are unused. The lab's "reducing metadata footprint" optimization places `next` and `prev` at the **end** of the metadata. This means:

- Allocated blocks: the user data starts after just `size`, `left_size`, and `allocated`.
- Free blocks: the full header including `next` and `prev` must fit.

The malloc-returned pointer always points to the beginning of user data (i.e., `sizeof(allocated_metadata)` bytes past the header start).

## Three Size Concepts

| Name | Definition | Used For |
|---|---|---|
| **request size** (raw_size) | What the user passed to `malloc()` | Input parameter |
| **actual size** | Total bytes the block occupies (data + all metadata) | Stored in `size` and `left_size` fields |
| **allocable size** | `actual_size - sizeof(allocated_metadata)` — the space available to the user | Indexing into `freelistSentinels` array |

The `size` and `left_size` fields always store the **actual size** (including metadata). Allocable sizes are just what's left for the user.

## Example from HW1 Solutions

Arena size = 128 bytes. After initialization:

| Block | Offset | Size (actual) | Left Size | Allocated |
|---|---|---|---|---|
| 1 | ...0000 | 88 | 16 | 0 (free) |
| 2 | ...0088 | 40 | 88 | 1 (allocated) |

Block 1 is free with actual size 88. Block 2 is the fencepost (dummy header at end of arena) with size 40 (header overhead). `left_size` of Block 1 is 16 (the size of the leading fencepost).

After `malloc(8)` on a heap with one free block of actual size 64:
- Block 1 splits: new block (size 32, allocated) at higher offset; remainder (size 32, free) at lower offset.
- Returns: `base_address + sizeof(BoundaryTag)`.

## Key Terms

| Term | Meaning |
|---|---|
| `size` | Actual total bytes this block occupies in memory |
| `left_size` | Actual size of the left (lower) neighbor block; enables O(1) left-neighbor access |
| `allocated` | Flag: 1=allocated, 0=free |
| `next` / `prev` | Free-list links; only valid when block is free |
| Actual size | Total block size including all metadata |
| Allocable size | Actual size minus allocated-metadata size; what user gets |
| Request size | Raw size argument passed to `malloc()` |
