## What Is a Premature Free?

From the slides:
> "A premature free is caused when an object that is still in use by the program is freed."

It is also called a **Use-After-Free** (UAF) error. The program frees memory but continues to hold a pointer to it and access it through that pointer.

## What Goes Wrong

When `free(p)` is called:
1. The block is added to the free list.
2. The allocator writes `next` and `prev` pointers **into the freed block's memory** (they overlay what was previously the user's data).

If the program then writes through `p`:

```c
int *p = (int*) malloc(sizeof(int));
*p = 8;
free(p);   // free updates next/prev in the block — these live where *p used to be

*p = 9;    // This overwrites the free-list's next pointer!
           // Future malloc/free will follow a corrupted pointer and crash.
```

The crash may happen **far from the location of the bug** — only when the corrupted pointer is eventually dereferenced by a future malloc or free call. This makes premature frees very difficult to debug.

## Security Implications

A use-after-free is also a **security vulnerability**: an attacker who can control what is written through a freed pointer can overwrite the `next`/`prev` pointers in the free list. This can be used to redirect future allocations to attacker-controlled memory regions.

## Mitigation: Set Pointer to NULL After Free

From the slides:
> "To minimize the impact of premature frees and double frees, set p = NULL after a free()."

```c
int *p = (int*) malloc(sizeof(int));
*p = 8;
free(p);
p = NULL;   // Now p can no longer be used accidentally

*p = 9;    // SEGV — crashes immediately at the source of the bug!
           // Now you can use gdb to determine there was a premature free.
```

Setting `p = NULL` causes any subsequent dereference to generate a SIGSEGV **immediately**, pointing gdb to the exact location of the use-after-free. Without the NULL assignment, the crash happens later and elsewhere, making root-cause analysis much harder.

## Key Terms

| Term | Meaning |
|---|---|
| Premature free | Freeing an object that is still in use |
| Use-After-Free (UAF) | Accessing memory after it has been freed |
| Free-list corruption | Overwriting next/prev pointers by writing to freed memory |
| `p = NULL` after free | Mitigation that causes immediate SEGV on subsequent dereference |
| Deferred crash | The crash from UAF happens later, far from the actual error line |
