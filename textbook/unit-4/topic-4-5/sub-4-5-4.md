## What Is a Double Free?

From the slides:
> "Double free is caused by freeing an object that is already free."

When `free(p)` is called on a block that is already in the free list:

1. The block is inserted into the free list a second time.
2. The free list now contains the same block **twice**.
3. The block's `next` and `prev` pointers get overwritten by the second insertion.
4. The free list's linked-list structure is corrupted.

From the slides:
> "This can cause the object to be added to the free list twice, corrupting the free list. After a double free, future calls to malloc/free may crash."

## Example

```c
int *p = (int*) malloc(sizeof(int));
// ...
free(p);   // Block added to free list; next/prev set correctly
// ...
free(p);   // Block inserted again — overwrites the next/prev ptr
           // from the first insert, corrupting the free list
           // Future malloc/free will crash
```

## Mitigation 1: Set p = NULL After Free

```c
free(p);
p = NULL;

free(p);   // free(NULL) is a no-op — safely ignored
           // or gives SEGV on NULL dereference — easier to debug
```

From the slides:
> "The second free will either be ignored or it will give SEGV so you can debug it."

## Mitigation 2: FREE() Macro

Some programmers define a safe macro that combines free and NULL-assignment:

```c
#define FREE(p) { free(p); p = NULL; }

// Usage:
int *x = (int*) malloc(sizeof(int));
// ...
FREE(x);   // frees and nulls in one step
```

## Magic Number Detection

The debugging allocator technique uses the **entire `allocated` field** (4 bytes) as a magic number rather than just a single bit:

- **Free**: `0xF7EEF7EE`
- **Allocated**: `0xA10CA7ED`

When `free(p)` is called, the allocator checks the block's `allocated` field:

```c
if (block->allocated == 0xF7EEF7EE) {
    // Block is already free — double free detected!
    // Crash with a useful error message
}
if (block->allocated != 0xA10CA7ED) {
    // Not a malloc-returned pointer — wild free detected!
}
```

This detects both double frees and wild frees (freeing a pointer not returned by malloc).

## Key Terms

| Term | Meaning |
|---|---|
| Double free | Calling `free()` on a pointer that has already been freed |
| Free-list corruption | Two copies of the same block in the list → corrupted next/prev pointers |
| `free(NULL)` | No-op — safe to call; does nothing |
| Magic number | 4-byte value in the `allocated` field used to distinguish free (`0xF7EEF7EE`) from allocated (`0xA10CA7ED`) blocks in a debugging allocator |
| `FREE(p)` macro | `{ free(p); p = NULL; }` — prevents double-free by nulling the pointer |
