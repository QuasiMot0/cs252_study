## External Fragmentation

**External fragmentation** is waste caused by free memory being split into many non-contiguous small blocks. Even if the total free memory is sufficient to satisfy a request, no single block is large enough.

### Formula

```
Ext.Frag% = 100 × (1 - size_largest_free_block / total_free_memory)
```

### Example (from slides)

Free blocks: 72 bytes + 64 bytes + 56 bytes = 192 bytes total free.

`malloc(100)` request: needs 100 bytes of data + 16 bytes header = 116, rounded to 120 bytes actual.

Largest single free block is 72 bytes — cannot satisfy 120 bytes.

```
Ext.Frag% = 100 × (1 - 72/192) = 100 × (120/192) ≈ 62.5%
```

### What Causes It

- **Splitting** large blocks into smaller ones.
- Freed blocks becoming stranded between allocated blocks.

### What Reduces It

- **Coalescing**: merging adjacent free blocks immediately on free.
- **Best-fit** placement: choosing the smallest block that fits, leaving less scattered leftover.

## Internal Fragmentation

**Internal fragmentation** is waste caused by the allocator returning more memory than the user requested. The extra bytes are inside the allocated block but are unusable by the caller.

### Formula

```
Int.Frag% = 100 × (1 - size_requested / mem_allocated)
```

### Example (from slides)

`malloc(25)`: actual_size = roundup8(25) + sizeof(header) = 32 + 8 = 40 bytes allocated.

```
Int.Frag% = 100 × (1 - 25/40) = 100 × (15/40) = 37.5%
```

### What Causes It

- **Alignment rounding**: requests are rounded up to multiples of 8.
- **Not splitting**: when the remainder after a potential split is too small, the entire block is used.

### What Reduces It

- Finer rounding granularity (smaller alignment requirement), at the cost of SIGBUS risk on RISC.
- Always splitting when possible (risks fragmentation with many tiny leftover blocks).

---

## Comparison

| Type | Caused By | Reduced By | Formula |
|---|---|---|---|
| External | Scattered free blocks, splitting | Coalescing, best-fit | 100 × (1 - largest_free / total_free) |
| Internal | Alignment rounding, no-split | Smaller alignment, split aggressively | 100 × (1 - requested / allocated) |

## Key Terms

| Term | Meaning |
|---|---|
| External fragmentation | Waste from free memory being non-contiguous |
| Internal fragmentation | Waste from allocating more bytes than requested |
| Coalescing | Merging adjacent free blocks; reduces external fragmentation |
| Alignment rounding | Cause of internal fragmentation (rounding to 8-byte boundary) |
