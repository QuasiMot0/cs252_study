## What Is a Wild Free?

From the slides:
> "Wild frees happen when a program attempts to free a pointer in memory that was not returned by malloc."

Wild frees are also called **"free of non-heap objects."** The freed pointer does not point to the beginning of a malloc'd block and therefore does not have a valid header immediately before it.

## Two Common Forms

### Form 1: Freeing a stack variable

```c
int q;
int *p = &q;   // p points to a stack variable, not heap memory

free(p);       // p points to memory without a malloc header
               // free will read garbage as a "header" → crash or corruption
```

### Form 2: Freeing an interior pointer (pointer arithmetic error)

```c
char *p = (char*) malloc(1000);
p = p + 10;   // p now points 10 bytes into the allocated block

free(p);      // p does not point to the start of the block
              // The bytes before p are not a valid header
              // free will crash or corrupt the free list
```

## Why It Crashes or Corrupts

When `free(p)` is called, the allocator looks for the block's header at `p - sizeof(allocated_metadata)`. For a wild-free pointer, that address contains garbage data — not a real header. When the allocator reads the garbage as `size`, `left_size`, and `allocated`, it will:

- Compute wrong neighbor addresses.
- Write `next`/`prev` pointers into wrong memory locations.
- Either crash immediately or corrupt the free list so a future malloc/free crashes.

From the slides:
> "Since the memory was not returned by malloc, it does not have a header. When attempting to free this non-heap object, the free may crash. Also if it succeeds, the free list will be corrupted so future malloc/free calls may crash."

## The malloc/new — free/delete Rule

From the slides:
> "Memory allocated with malloc() should only be deallocated with free() and memory allocated with new() should only be deallocated with delete()."

Mixing these is also a form of wild free: `delete` expects a C++ object header; `free` expects a malloc header. They are not interchangeable.

## Detection via Magic Numbers

The debugging allocator detects wild frees by checking the magic number in the `allocated` field:

```c
if (block->allocated != 0xA10CA7ED) {
    // Not a valid malloc block — likely a wild free
    // Report the error
}
```

## Key Terms

| Term | Meaning |
|---|---|
| Wild free | Freeing a pointer not returned by `malloc()` |
| Free of non-heap object | Freeing a stack variable or global — another name for wild free |
| Interior pointer | A pointer into the middle of an allocated block (e.g., after `p = p + 10`) |
| malloc/free pairing | `malloc` must be paired with `free`; `new` must be paired with `delete` |
