## What Are Segregated Free Lists?

Instead of one free list for all sizes, segregated free lists maintain **multiple lists**, one per size class. When the allocator needs a block of a particular size, it jumps directly to the list for that size class rather than scanning all free blocks.

For a traditional segregated list (BSD UNIX approach), size classes might be: 8, 16, 32, 64, 128, 256, ≥512 bytes.

## The CS252 Lab Implementation

The lab uses `N_LISTS` lists indexed 0 through `N_LISTS-1`, organized by **allocable size** (the size available to the user, not including the header metadata). The lists are ordered in multiples of 8:

| Index | Min Allocable | Max Allocable |
|---|---|---|
| 0 | 1 | 8 |
| 1 | 9 | 16 |
| 2 | 17 | 24 |
| … | … | … |
| N_LISTS-2 | (N_LISTS-2)×8+1 | (N_LISTS-1)×8 |
| N_LISTS-1 | (N_LISTS-1)×8+1 | and up (unbounded) |

**Index 0 is always empty** in the CS252 implementation. On a 64-bit system, the minimum allocable size is 16 bytes (because the free block must hold `next` and `prev` pointers, each 8 bytes). On 32-bit, the minimum is 8 bytes due to alignment. Either way, index 0 (max 8 bytes) is never used.

## freelistSentinels Array

The array `freelistSentinels[N_LISTS]` stores a sentinel node for each list. A sentinel is a header that contains `next` and `prev` pointers but no allocable memory — it anchors the list.

```
freelistSentinels[0] → (always empty)
freelistSentinels[1] → [Block A (16 bytes)] → [Block B (16 bytes)] → (back to sentinel)
freelistSentinels[2] → (empty)
freelistSentinels[3] → [Block C (24 bytes)] → [Block D (24 bytes)] → (back to sentinel)
...
freelistSentinels[N_LISTS-1] → [Block E (1024)] → [Block F (512)] → [Block G (1000)] → (back)
```

## Circular Doubly-Linked Lists

Each list is a **circular doubly-linked list**:

- The sentinel's `prev` points to the last node; the sentinel's `next` points to the first node.
- An empty list has both `sentinel->next` and `sentinel->prev` pointing back to the sentinel itself.
- All insertions are made at the **beginning** of the list (just after the sentinel).

## Finding the Right List

To find which list to search for a block of allocable size `s`:

```
index = s / 8;          // integer division
if (index >= N_LISTS)
    index = N_LISTS - 1; // clamp to last list
```

When the desired list is empty, check the next list up, and so on. The last list (index `N_LISTS-1`) uses first-fit since blocks there vary in size.

## Key Terms

| Term | Meaning |
|---|---|
| Segregated free list | Multiple free lists, one per size class |
| `freelistSentinels` | Array of sentinel nodes, one per size class |
| Allocable size | Block size minus allocated metadata; used as list index |
| Size class | Range of allocable sizes mapped to one list (multiples of 8) |
| Index 0 always empty | Minimum allocable size is 16 bytes (64-bit) or 8 bytes (32-bit), so index 0 never holds blocks |
