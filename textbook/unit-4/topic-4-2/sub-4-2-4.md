## First-Fit

**First-fit** allocates the **first** free block that is large enough to satisfy the request.

- Walk the free list from the beginning.
- Use the first block that fits; stop searching.
- **Faster** — stops as soon as it finds a match.
- **Downside**: if always starting from the front, small leftover chunks accumulate at the beginning of the list over time, causing the allocator to scan past many small blocks before finding a large one.

**Next-fit** is a variant: start searching from where the last search ended rather than the beginning. This distributes leftover chunks more evenly.

## Best-Fit

**Best-fit** allocates the **smallest** free block that is large enough to satisfy the request.

- Walk the entire free list to find the minimum-sized block that fits.
- **Less external fragmentation** — wastes less space in the leftover remainder.
- **Slower** — must scan the entire list every time.

## Comparison

| Policy | Speed | External Fragmentation | Notes |
|---|---|---|---|
| First-fit | Faster (stops early) | More | Small chunks cluster at list front |
| Next-fit | Medium | Medium | Distributes chunks more evenly |
| Best-fit | Slower (full scan) | Less | Minimizes leftover waste |

From the slides: "First fit is faster but best fit uses less space."

## CS252 Lab Implementation

With segregated free lists, most lists hold blocks of uniform size. For lists 0 through `N_LISTS-2`, every block in the list is exactly the right size for the requested allocation class — so there is no search at all; the allocator just takes the first (front) block.

For the **last list** (`N_LISTS-1`), blocks vary in size. The CS252 specification requires **first-fit** there: iterate over the blocks and use the first block large enough to satisfy the request.

The block allocated to the user comes from the **smallest list that can satisfy the request**. When the desired list is empty, the next non-empty list is checked.

## Key Terms

| Term | Meaning |
|---|---|
| First-fit | Use the first free block that is large enough |
| Best-fit | Use the smallest free block that is large enough |
| Next-fit | First-fit starting from where the last search ended |
| Last list (`N_LISTS-1`) | The catch-all list for large blocks; uses first-fit |
