## What Is an Explicit Free List?

An **explicit free list** adds `next` and `prev` pointers to each free block's header, linking all free blocks together in a doubly-linked list. The allocator only has to traverse **free blocks** — allocated blocks are skipped entirely.

```
Sentinel ↔ Free Block A ↔ Free Block B ↔ Free Block C ↔ (back to Sentinel)
```

The `next` and `prev` pointers live in the block's metadata. For an **allocated** block, those fields are not needed — the block is not in any free list. For a **free** block, those fields form the links in the free list.

## Metadata Layout

```
| size | left_size | allocated | [next] | [prev] | ... user data ... |
```

- **allocated blocks**: only `size`, `left_size`, and `allocated` are used.
- **free blocks**: `next` and `prev` are also used to link the block into the free list.

## How Allocation Works

1. Start from the sentinel node.
2. Walk via `next` pointers through the free list.
3. Find the first (or best) block large enough.
4. Remove that block from the list by relinking its neighbors: `prev->next = block->next; block->next->prev = block->prev`.
5. Split if needed, mark as allocated.

## How Free Works

1. Mark block as unallocated.
2. Insert block into the free list (at the front, i.e., just after the sentinel).
3. Coalesce with adjacent free neighbors if applicable (requires checking physical neighbors, not free-list neighbors).

## Key Advantage Over Implicit List

Allocation is O(f) where f = number of **free** blocks, not total blocks. For heaps with many allocated blocks, this is significantly faster.

## Sentinel Node

A **sentinel** (also called a dummy head node) is a special header that always exists at the start of the free list. It never holds user data. It simplifies insert/remove code by eliminating edge cases for empty lists or operations at the list boundaries.

## Key Terms

| Term | Meaning |
|---|---|
| Explicit free list | Doubly-linked list containing only free blocks |
| `next` / `prev` | Pointers stored in free block's header linking it to the list |
| Sentinel | Dummy head node that anchors the circular list |
| O(f) allocation | Traverse only f free blocks, not all n total blocks |
