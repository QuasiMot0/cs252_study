## Pointer Arithmetic

When you add an integer to a pointer, the pointer advances by `integer × sizeof(pointed-to type)` bytes:

From the slides:
> "When you increment a pointer to integer it will be incremented by 4 units because sizeof(int)==4."

```c
int *p = &array[0];
p++;    // p now points to array[1]; address increases by 4 (sizeof int)

char *c = (char *)array;
c++;    // c advances by 1 byte (sizeof char)
```

Pointer comparison is also valid: `p < pend` tests whether `p` has not reached the end.

## Array-Pointer Equivalence

From the slides, all of the following are equivalent for `int a[20]`:

| Expression | Meaning |
|---|---|
| `a[i]` | Standard array indexing |
| `*(a+i)` | Pointer arithmetic + dereference |
| `*(&a[0]+i)` | Explicit address of first element + offset |
| `*((int*)((char*)&a[0]+i*sizeof(int)))` | Byte-level pointer arithmetic |

From the slides:
> "C was designed to be machine independent assembler."

The last form shows what the hardware actually does: compute a byte address, cast to the element type, dereference.

## Using Pointers to Optimize Array Traversal

From the slides, the indexed version:
```c
int sum(int *array, int n) {
    int s = 0;
    for (int i = 0; i < n; i++) {
        s += array[i];   // equivalent to *(int*)((char*)array + i*sizeof(int))
    }
    return s;
}
```

Can be rewritten with pointers to avoid repeated multiplication:
```c
int sum(int *array, int n) {
    int s = 0;
    int *p = &array[0];
    int *pend = &array[n];
    while (p < pend) {
        s += *p;
        p++;     // advances by sizeof(int) = 4 bytes
    }
    return s;
}
```

From the slides:
> "Using pointers is more efficient because no indexing is required and indexing require multiplication."
> "Note: An optimizer may substitute the multiplication by a '<<' operator if the size is a power of two. However, the array entries may not be a power of 2 and integer multiplication may be needed."

## Key Terms

| Term | Meaning |
|---|---|
| Pointer arithmetic | `p + i` advances by `i * sizeof(*p)` bytes |
| Array-pointer equivalence | `a[i]` == `*(a+i)` == `*(&a[0]+i)` |
| `p++` on `int*` | Advances pointer by 4 bytes (sizeof int) |
| `pend = &array[n]` | Points one past the last element; used as loop sentinel |
| Indexed vs pointer loop | Pointer loop avoids per-iteration multiplication; more cache-friendly |
