## Three Implementations of 2D Arrays

The slides present three different ways to implement a 2D array in C.

---

## 1st Implementation: Normal (Contiguous) 2D Array

```c
int a[4][3];
```

- All 12 elements are stored contiguously in memory in **row-major order**.
- Memory layout for `a[4][3]` starting at address 100:
  - `a[0][0]`:100, `a[0][1]`:104, `a[0][2]`:108
  - `a[1][0]`:112, `a[1][1]`:116, `a[1][2]`:120
  - `a[2][0]`:124, `a[2][1]`:128, `a[2][2]`:132
  - `a[3][0]`:136, `a[3][1]`:140, `a[3][2]`:144

The address formula:
```
a[i][j] == *(int*)((char*)a + i*3*sizeof(int) + j*sizeof(int))
```

---

## 2nd Implementation: Array of Pointers to Rows (Jagged, Stack)

```c
int *(a[4]);   // a is an array of 4 pointers to int

for (int i = 0; i < 4; i++) {
    a[i] = (int*)malloc(sizeof(int) * 3);
    assert(a[i] != NULL);
}
```

- `a` is on the stack; it holds 4 pointers.
- Each `a[i]` points to a separately heap-allocated row of 3 ints.
- The rows are **not** contiguous in memory.
- Access: `a[3][2] = 5` — works the same syntactically.

---

## 3rd Implementation: Pointer to Array of Pointers (Fully Dynamic)

```c
int **a;
a = (int**)malloc(4 * sizeof(int*));
assert(a != NULL);

for (int i = 0; i < 4; i++) {
    a[i] = (int*)malloc(3 * sizeof(int));
    assert(a[i] != NULL);
}
```

- `a` itself is a heap pointer to an array of 4 `int*` pointers.
- Each `a[i]` points to a separately heap-allocated row.
- Access: `a[3][2] = 5` — identical syntax; two levels of indirection.

---

## Advantages of Pointer-Based Arrays

From the slides:
> "You don't need to know in advance the size of the array (dynamic memory allocation)."
> "You can define an array with different row sizes."

Example: **Triangular matrix** using `int **a`:
- Row 0: 4 elements (`a[0][0..3]`)
- Row 1: 3 elements (`a[1][0..2]`)
- Row 2: 2 elements (`a[2][0..1]`)
- Row 3: 1 element (`a[3][0]`)

Each row is allocated with `malloc(i * sizeof(int))` — impossible with a fixed 2D array.

## Comparison

| Feature | `int a[4][3]` | `int*(a[4])` | `int **a` |
|---|---|---|---|
| Where is `a`? | Stack | Stack | Heap |
| Row allocation | Automatic | Heap (malloc) | Heap (malloc) |
| Rows contiguous? | Yes | No | No |
| Size known at compile time? | Yes | Yes | No |
| Variable row sizes? | No | No | Yes |

## Key Terms

| Term | Meaning |
|---|---|
| Row-major order | 2D array stored row by row in contiguous memory |
| Jagged array | 2D array where rows are separately allocated and not contiguous |
| `int **a` | Pointer to an array of `int*` pointers; fully dynamic 2D array |
| Triangular matrix | Jagged array where rows have different lengths |
