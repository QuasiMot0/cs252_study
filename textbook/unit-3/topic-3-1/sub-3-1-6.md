## Rule: Always Point to Valid Memory

From the slides:
> "When using pointers make sure the pointer is pointing to valid memory before assigning or getting any value from the location."

Using a pointer before it points to valid memory is one of the most common C bugs.

## Error 1: Uninitialized Pointer

A declared-but-uninitialized pointer contains garbage — some random address that is almost certainly invalid.

```c
char *s;
strcpy(s, "hello");   // SEGV — uninitialized pointer
```

From the slides:
> "String functions do not allocate memory for you: `char *s; strcpy(s, "hello"); --> SEGV(uninitialized pointer)`"

**Fix**: Initialize the pointer before use:
```c
char *s = malloc(strlen("hello") + 1);
strcpy(s, "hello");
```

## Error 2: strdup — The Only String Function That Allocates

From the slides:
> "The only string function that allocates memory is strdup (it calls malloc of the length of the string and copies it)."

```c
char *s = strdup("hello");   // allocates + copies; s is ready to use
// ...
free(s);   // must free when done
```

All other string functions (`strcpy`, `strcat`, `sprintf`, etc.) require the destination buffer to already be allocated.

## Error 3: Null Pointer Dereference

Dereferencing a NULL pointer causes an immediate SEGV:

```c
int *p = NULL;
*p = 5;   // SEGV
```

Always check pointers returned by `malloc`/`fopen`/etc.:
```c
int *p = (int*)malloc(sizeof(int));
if (p == NULL) {
    // handle allocation failure
}
```

## Error 4: Dangling Pointer

A dangling pointer points to memory that has already been freed:

```c
int *p = (int*)malloc(sizeof(int));
free(p);
*p = 5;   // undefined behavior — p is now dangling
```

**Fix**: Set the pointer to NULL after freeing:
```c
free(p);
p = NULL;
```

## Printing Pointers for Debugging

From the slides:
> "It is useful to print pointers for debugging."

```c
char buff[10];

printf("ptr=%ld\n", (long)&buff[5]);               // as decimal
printf("ptr=0x%lx\n", (unsigned long)&buff[5]);    // as hex
printf("ptr=%p\n", &buff[5]);                       // portable format
```

From the slides:
> "Instead of using printf(), I recommend to use fprintf(stderr, ...) since stderr is unbuffered and it is guaranteed to be printed on the screen."

`fprintf(stderr, ...)` is preferred because stdout may be buffered — if the program crashes before flushing, you lose the output.

## Key Terms

| Term | Meaning |
|---|---|
| Uninitialized pointer | Declared pointer with no assigned value; contains garbage; dereference causes SEGV |
| Null pointer dereference | `*NULL` — immediate SEGV |
| Dangling pointer | Pointer to freed memory; dereference is undefined behavior |
| `strdup` | Only standard string function that allocates memory; must be freed |
| `fprintf(stderr,...)` | Preferred for debug prints; stderr is unbuffered so output appears before crash |
