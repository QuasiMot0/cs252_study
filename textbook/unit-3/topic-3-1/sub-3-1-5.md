## What Are Function Pointers?

From the slides:
> "Pointers to functions are often used to implement Polymorphism in 'C'."
> "**Polymorphism**: Being able to use the same function with arguments of different types."

A function pointer stores the address of a function, allowing it to be called indirectly. This enables passing behavior as a parameter — the C equivalent of callbacks.

## Declaration and typedef

Raw function pointer syntax is hard to read. `typedef` creates a named type:

```c
typedef void (*FuncPtr)(int a);
// FuncPtr is a type: pointer to a function taking int, returning void
```

Without typedef:
```c
void (*fp)(int a);   // fp is a variable: pointer to a function taking int, returning void
```

From the slides:
> "FuncPtr is a type of a pointer to a function that takes an 'int' as an argument and returns 'void'."

## Array Mapper Example

A function that applies any `FuncPtr` to each element of an array:

```c
typedef void (*FuncPtr)(int a);

void intArrayMapper(int *array, int n, FuncPtr func) {
    for (int i = 0; i < n; i++) {
        (*func)(array[i]);
    }
}

int s = 0;
void sumInt(int val)   { s += val; }
void printInt(int val) { printf("val = %d\n", val); }
```

Usage:
```c
int a[] = {3, 4, 7, 8};

// Print all elements
intArrayMapper(a, sizeof(a)/sizeof(int), printInt);

// Sum all elements
s = 0;
intArrayMapper(a, sizeof(a)/sizeof(int), sumInt);
printf("total=%d\n", s);
```

## Generic Mapper (void*)

To work with any element type, use `void*` and pass the element size:

```c
typedef void (*GenFuncPtr)(void *a);

void genericArrayMapper(void *array, int n, int entrySize, GenFuncPtr fun) {
    for (int i = 0; i < n; i++) {
        void *entry = (void*)((char*)array + i * entrySize);
        (*fun)(entry);
    }
}

void sumIntGen(void *pVal) {
    int *pInt = (int*)pVal;
    s += *pInt;
}
void printIntGen(void *pVal) {
    int *pInt = (int*)pVal;
    printf("Val = %d\n", *pInt);
}
```

Usage:
```c
int a[] = {3, 4, 7, 8};
genericArrayMapper(a, sizeof(a)/sizeof(int), sizeof(int), printIntGen);
genericArrayMapper(a, sizeof(a)/sizeof(int), sizeof(int), sumIntGen);
```

## Generic Sort (mysort)

From the slides, a generic sort uses a comparison function pointer:

```c
mysort(void *array, int ascending, int n, int elemSize, ComFunc comp)
// Assume bubblesort
for (int j = 0; ...) {
    for (int i = 0; ...) {
        char *p = (char*)array + i * elemSize;         // address of element i
        char *q = (char*)array + (i+1) * elemSize;    // address of element i+1
        int result = comp((void*)p, (void*)q);
        // Swap if necessary
    }
}
```

## Swapping Generic Elements

From the slides, to swap two elements of any type using `memcpy`:

```c
void *tmp = (void *)malloc(entrySize);
assert(tmp != NULL);
memcpy(tmp, a, entrySize);
memcpy(a, b, entrySize);
memcpy(b, tmp, entrySize);
```

> "Note: You may allocate memory only once for tmp in the sort method and use it for all the sorting to save multiple calls to malloc. Free tmp at the end."

## String Comparison in Sort

When sorting an array of strings (`char*` entries), the comparison function receives a pointer to a `char*` (i.e., a `char**`):

```c
int StrComFun(void *pa, void *pb) {
    char **stra = (char**)pa;
    char **strb = (char**)pb;
    return strcmp(*stra, *strb);
}
```

## Key Terms

| Term | Meaning |
|---|---|
| Function pointer | Variable holding the address of a function; called via `(*fp)(args)` |
| `typedef void (*FuncPtr)(int)` | Creates `FuncPtr` as a named type for a function pointer |
| Polymorphism | Using the same function interface with arguments of different types |
| `GenFuncPtr` | Function pointer taking `void*`; works with any element type |
| `memcpy(dst, src, n)` | Copies n bytes from src to dst; used for generic element swap |
| `ComFunc` | Comparison function pointer: takes two `void*`, returns int (like `strcmp`) |
