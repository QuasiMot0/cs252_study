## What Is a System Call?

From the slides:
> "System Calls is the way user programs request services from the OS"
> "System calls use Software Interrupts"
> "System calls is the API of the OS from the user program's point of view. See /usr/include/sys/syscall.h"

Examples from the slides:
```c
open(filename, mode)
read(file, buffer, size)
write(file, buffer, size)
fork()
execve(cmd, args);
```

## Why Software Interrupts Instead of Function Calls?

From the slides:
> "Software Interrupts will switch into kernel mode"
> "OS services need to run in kernel mode because:"
> "They need privileged instructions"
> "Accessing devices and kernel data structures"
> "They need to enforce the security in kernel mode."

A regular function call stays in user mode and cannot access hardware or kernel structures. A software interrupt forces a mode switch into kernel mode, where the OS can safely execute the requested service.

## The Syscall Mechanism (libc wrapper → software interrupt)

From the slides:
> "Libc (the C library) provides wrappers for the system calls that eventually generate the system calls."

```c
// User Mode (libc wrapper):
int open(fname, mode) {
    return syscall(SYS_open, fname, mode);
}

int syscall(syscall_num, …) {
    asm(INT);    // software interrupt
}
```

```
// Kernel Mode (syscall interrupt handler):
switch (syscall_num):
  SYS_READ: …
  SYS_WRITE: …
  SYS_OPEN:
    - Get file name and mode
    - Check if file exists
    - Verify permissions of file against mode.
    - Ask disk to perform operation. Make process wait until operation is complete
    - return fd (file descriptor)
```

## Security Enforcement in the Handler

From the slides:
> "The software interrupt handler for system calls has entries for all system calls."
> "The handler checks that the arguments are valid and that the operation can be executed."
> "The arguments of the syscall are checked to enforce the security and protections."

For `open(filename, mode)`, the handler verifies:
- If file does not exist → return error
- If permissions of file do not agree with the mode → return error (also considers owner of the file and owner of the calling process)
- If all checks pass → open file and return file descriptor

For `write(fd, buff, n)`, the handler verifies:
- `fd` is a file descriptor opened in write mode
- `[buff, buff+n-1]` is a valid memory range
- If any check fails → return -1 and set `errno`

## Syscall Error Reporting

From the slides:
> "When an error in a system call occurs, the OS sets a global variable called 'int errno' defined in libc.so with the number of the error that gives the reason for failure."
> "The list of all the errors can be found in /usr/include/sys/errno.h"

```c
#define EPERM   1   /* Not super-user     */
#define ENOENT  2   /* No such file or directory */
#define ESRCH   3   /* No such process    */
#define EINTR   4   /* interrupted system call */
#define EIO     5   /* I/O error          */
#define ENXIO   6   /* No such device or address */
```

> "You can print the corresponding error message to stderr using `perror(s);` where s is a string prepended to the message."

## What Is and Is Not a System Call

From the slides:
> "Only operations that need to be executed by the OS in kernel mode are part of the system calls."
> "Function like sin(x), cos(x) are not system calls."
> "Some functions like printf(s) run mainly in user mode but eventually call write() when the buffer is full and needs to be flushed."
> "Also malloc(size) will run mostly in user mode but eventually it will call sbrk() to extend the heap."

## strace: Observing System Calls

From the slides:
> "It is a tool that prints the system calls that a program calls."

```bash
$ strace ./hello
execve("./hello", ["./hello"], [/* 40 vars */]) = 0
brk(0)                              = 0x2520000
mmap(NULL, 4096, PROT_READ|PROT_WRITE, ...)
write(1, "Hello cs252\n", 12)       = 12
write(1, "Hello again\n", 12)       = 12

```

## Key Terms

| Term | Meaning |
|---|---|
| System call | Mechanism for user programs to request OS services; uses software interrupt |
| Software interrupt (`INT`) | Assembly instruction that switches CPU to kernel mode |
| libc wrapper | User-mode function in libc that sets up args and triggers the software interrupt |
| Syscall table | Array in `/usr/include/sys/syscall.h` mapping names to integers |
| `errno` | Global variable set by OS to indicate the reason a system call failed |
| `perror(s)` | Prints the error string for the current `errno` value, prefixed by `s` |
| `strace` | Tool that traces and prints all system calls made by a program |
| `sbrk()` | System call to extend the heap (called internally by `malloc`) |
