## Classifying Variables by Section

The section where a variable is stored depends on how it is declared:

| Declaration | Section | Reason |
|---|---|---|
| `int a = 5;` (global) | **Data** | Initialized global variable |
| `int b[20];` (global, no initializer) | **BSS** | Uninitialized global; zeroed by OS |
| `int main() { ... }` | **Text** | Function code is instructions |
| `int x;` (inside a function) | **Stack** | Local variable; automatic lifetime |
| `malloc(100)` | **Heap** | Explicitly allocated dynamic memory |

From the slides:
> "Example: `int a=5`→data; `int b[20]`→bss; `int main()`→text; `int x`→stack; `malloc()`→heap"

## Example Program

```c
int a = 5;          // Data section (initialized global)
int b[20];          // BSS section (uninitialized global, zero-filled)

int main() {        // Text section (code)
    int x;          // Stack (local variable)
    int *p = malloc(sizeof(int) * 10);  // p on stack, allocation on Heap
    return 0;
}
```

## Why It Matters

- **BSS** variables are always initialized to zero (you can rely on this).
- **Stack** variables are NOT initialized — reading them before writing gives garbage.
- **Heap** memory from `malloc` is NOT initialized (use `calloc` for zeroed allocation).
- **Global** variables persist for the entire program lifetime; **stack** variables disappear when the function returns.

## Key Terms

| Term | Meaning |
|---|---|
| Data section | Holds initialized globals (`int a = 5;`) |
| BSS section | Holds uninitialized globals; zero-filled by the OS at program start |
| Stack variable | Local variable; exists only while its function is on the call stack |
| Heap allocation | Created by `malloc`/`new`; persists until explicitly freed |
| Automatic vs dynamic lifetime | Stack = automatic (compiler manages); Heap = dynamic (programmer manages) |
