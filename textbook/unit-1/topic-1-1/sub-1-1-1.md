## The Virtual Address Space

Each program sees memory as a contiguous array from address `0` to `2^64 - 1` (0x0000000000000000 to 0xFFFFFFFFFFFFFFFF). This is the program's **virtual address space** — it is an illusion maintained by the OS; physical memory is allocated on demand.

## Memory Sections (Low to High)

| Section | Contents | Permissions | Notes |
|---|---|---|---|
| **Text** | Machine instructions | Read + Execute | Your code; read-only at runtime |
| **Data** | Initialized global/static vars | Read + Write | `int a = 5;` at global scope |
| **BSS** | Uninitialized global/static vars | Read + Write | Zeroed by OS at load time |
| **Heap** | Dynamic allocations (`malloc`/`new`) | Read + Write | Grows upward |
| **Shared Libs** | Mapped `.so` / `.dylib` files | Read + Execute | Shared across processes |
| **Stack** | Local variables, return addresses | Read + Write | Grows downward |

From the slides:
> "Program sees memory as an array from 0 to 2^64-1. Different sections have different permissions."

## Key Terms

| Term | Meaning |
|---|---|
| Text section | Stores compiled machine instructions; read + execute permissions |
| Data section | Stores initialized global/static variables |
| BSS | Block Started by Symbol; uninitialized globals; zeroed by OS |
| Heap | Dynamic memory region; grows upward; managed by `malloc`/`free` |
| Stack | Automatic memory region; grows downward; holds locals and return addresses |
| Shared libs | Dynamically linked libraries mapped into the address space at load time |
