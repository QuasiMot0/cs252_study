## What Are Registers?

Registers are the CPU's fastest storage — all arithmetic and logic operations use them. The x86-64 architecture has 16 general-purpose 64-bit registers.

From the slides:
> "All operations go through registers; operands, results, temporals; fetch/store from memory; pass args; return results."

## Register Table

| Register | Role | Preserved? |
|---|---|---|
| `%rax` | 1st return value; temp | No |
| `%rbx` | Callee-saved | Yes |
| `%rcx` | 4th argument | No |
| `%rdx` | 3rd argument; 2nd return value | No |
| `%rsi` | 2nd argument | No |
| `%rdi` | 1st argument | No |
| `%rbp` | Frame pointer; callee-saved | Yes |
| `%rsp` | Stack pointer | Yes |
| `%r8` | 5th argument | No |
| `%r9` | 6th argument | No |
| `%r10` | Temporary | No |
| `%r11` | Temporary | No |
| `%r12`–`%r14` | Callee-saved | Yes |
| `%r15` | Callee-saved / GOT pointer | Yes |

## Argument Passing Convention

The first 6 integer/pointer arguments are passed in registers (in order):

```
1st arg → %rdi
2nd arg → %rsi
3rd arg → %rdx
4th arg → %rcx
5th arg → %r8
6th arg → %r9
```

Additional arguments beyond the 6th go on the stack.

## Callee-Saved vs Caller-Saved

- **Callee-saved** (`%rbx`, `%rbp`, `%rsp`, `%r12`–`%r15`): If a function uses these, it must save them (via `push`) at entry and restore them (via `pop`) before returning. The caller can assume they are unchanged after a function call.
- **Caller-saved** (all others): These may be overwritten by any function call. If the caller needs their values preserved, it must save them before the call.

From the slides:
> "Arg registers may be modified by function calls (save with push/pop if needed); callee-saved must be saved at function start and restored before return."

## Example: Simple Function

```asm
sum:
    pushq  %rbp          # save callee-saved frame pointer
    movq   %rsp, %rbp   # set up frame
    movq   %rdi, %rax   # 1st arg → return register
    addq   %rsi, %rax   # add 2nd arg
    leave               # restore %rbp, %rsp
    ret                 # return (value in %rax)
```

## Key Terms

| Term | Meaning |
|---|---|
| Callee-saved register | Must be preserved across function calls; function saves/restores them |
| Caller-saved register | May be clobbered by a called function; caller saves if needed |
| Argument registers | `%rdi`, `%rsi`, `%rdx`, `%rcx`, `%r8`, `%r9` — first 6 integer args |
| Return register | `%rax` — holds the function's return value |
| `%rsp` | Stack pointer; always points to the top of the stack |
