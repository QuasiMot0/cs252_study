## Global Variables in Assembly

In C, a global variable is declared outside any function. In assembly, global variables are declared using assembler **directives** that place the variable in the Data or BSS section.

From the slides:
> "Global variables: `.data; .comm <var-name>, <data-size>[,<alignment>]`; e.g. `.comm a,8` for `long a;`"

## The `.comm` Directive

`.comm` declares a symbol in the **BSS section** (uninitialized common block). It is zero-filled by the OS at program start.

```asm
.data
.comm a, 8         # long a;       (8 bytes, BSS)
.comm b, 160       # long b[20];   (20 * 8 = 160 bytes, BSS)
```

Syntax: `.comm <symbol_name>, <size_in_bytes>[, <alignment>]`

## Byte-Level Register Access

x86-64 registers can be accessed in multiple widths. The lowest byte of each register has a separate name:

| Full 64-bit | Low 8-bit | Used with |
|---|---|---|
| `%rax` | `%al` | `movb`, `movsbq`, `movzbq` |
| `%rbx` | `%bl` | |
| `%rcx` | `%cl` | |
| `%rdx` | `%dl` | |
| `%r8` | `%r8b` | |

When working with `char` (1 byte), change the instruction suffix from `q` to `b`:
```asm
movb  (%rdi), %al   # load one byte (char)
movzbq %al, %rax    # zero-extend to 64 bits for further arithmetic
```

## Assembling and Running

From the slides:
> "`gcc -static -g -o t1 t1.s` then `./t1`"

Steps to assemble and run a `.s` file directly:
```bash
gcc -static -g -o t1 t1.s   # assemble + link to static executable
./t1                          # run it
```

The `-g` flag preserves debug symbols so `gdb` can inspect the file. The `-static` flag links all libraries statically.

## Executable File Formats

| Format | Platform |
|---|---|
| ELF (Executable and Linkable Format) | Linux, Solaris |
| COFF (Common Object File Format) | Windows |
| a.out | BSD / early UNIX (obsolete) |

From the slides:
> "ELF (Linux/Solaris), COFF (Windows), a.out (BSD/early UNIX, obsolete)"

## Key Terms

| Term | Meaning |
|---|---|
| `.data` directive | Marks the start of the data section in an assembly file |
| `.comm` | Declares an uninitialized global symbol in BSS with given size |
| Byte register | Lower 8-bit sub-register (`%al`, `%bl`, `%r8b`); use with `b` suffix instructions |
| `movzbq` | Move byte and zero-extend to quad (64-bit) |
| ELF | Executable and Linkable Format; standard binary format on Linux |
