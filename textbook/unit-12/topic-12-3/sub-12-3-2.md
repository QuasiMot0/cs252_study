# CREATE TABLE: Column Types and Constraints

## Basic Syntax

```sql
CREATE TABLE tablename (
    column1  datatype  [constraints],
    column2  datatype  [constraints],
    ...
);
```

## Common Data Types

### String Types

| Type | Description | Use case |
|------|-------------|----------|
| `CHAR(n)` | Fixed-length string, always n chars (pads with spaces) | State codes ('CA'), fixed codes |
| `VARCHAR(n)` | Variable-length string, up to n chars | Names, emails, addresses |
| `TEXT` | Large variable-length string | Blog posts, descriptions |

**Rule of thumb:** Use `CHAR` when data is always exactly n characters; use `VARCHAR` when length varies.

### Numeric Types

| Type | Description | Use case |
|------|-------------|----------|
| `INT` | Integer | IDs, counts, ages |
| `DECIMAL(p, s)` | Exact decimal: p total digits, s after decimal | Money, prices |
| `FLOAT` / `DOUBLE` | Approximate floating point | Scientific measurements |

`DECIMAL(8,2)` stores up to 999999.99 — 8 total digits, 2 after the decimal. Use for currency (never use FLOAT for money due to rounding errors).

### Date/Time Types

| Type | Description | Example |
|------|-------------|---------|
| `DATE` | Calendar date, no time | '2026-05-01' |
| `DATETIME` | Date + time | '2026-05-01 14:30:00' |
| `TIMESTAMP` | Date + time, auto-updates | Same format |

## Common Constraints

| Constraint | Meaning |
|-----------|---------|
| `NOT NULL` | Column cannot be NULL (required field) |
| `PRIMARY KEY` | Unique, not null, table's main identifier |
| `AUTO_INCREMENT` | Automatically assign next integer (MySQL-specific) |
| `UNIQUE` | No two rows can have the same value |
| `DEFAULT val` | Use val if no value is provided on INSERT |

## Full Example

```sql
CREATE TABLE employees (
    id        INT           PRIMARY KEY AUTO_INCREMENT,
    name      VARCHAR(100)  NOT NULL,
    dept      CHAR(2),
    salary    DECIMAL(10,2) DEFAULT 0.00,
    hire_date DATE          NOT NULL
);
```

- `id` — auto-incrementing integer primary key
- `name` — required, up to 100 chars
- `dept` — optional 2-char department code
- `salary` — up to 10 digits with 2 decimal places, defaults to 0
- `hire_date` — required date

## Another Example: Orders Table

```sql
CREATE TABLE orders (
    order_id    INT          PRIMARY KEY AUTO_INCREMENT,
    customer_id INT          NOT NULL,
    total       DECIMAL(8,2) NOT NULL,
    order_date  DATE         NOT NULL,
    status      VARCHAR(20)  DEFAULT 'pending'
);
```

## Choosing the Right Type

| Data | Best type |
|------|-----------|
| Person's name | `VARCHAR(100)` |
| US state code | `CHAR(2)` |
| Price / money | `DECIMAL(10,2)` |
| Row count / ID | `INT` |
| Birthday | `DATE` |
| Password hash | `CHAR(64)` (SHA-256 is always 64 hex chars) |

## Key Takeaways

- `CHAR(n)` = fixed length; `VARCHAR(n)` = variable length up to n
- `DECIMAL(p,s)` = p total digits, s after decimal — use for money
- `DATE` = date only; `DATETIME` = date + time
- `NOT NULL` makes a field required
- `PRIMARY KEY` = unique + not null; often combined with `AUTO_INCREMENT`
