# INSERT, UPDATE, and DELETE

## INSERT INTO

Adds a new row to a table.

### With column list (recommended):
```sql
INSERT INTO employees (name, dept, salary)
VALUES ('Dave', 'CS', 71000);
```
The column list and value list must correspond positionally. Columns not listed get their default value (or NULL if no default).

### Without column list:
```sql
INSERT INTO employees VALUES (4, 'Dave', 'CS', 71000);
```
Values must be supplied for **every** column in the order they were defined. Fragile — breaks if you add columns.

### Multiple rows at once:
```sql
INSERT INTO employees (name, dept, salary) VALUES
  ('Eve', 'HR', 52000),
  ('Frank', 'CS', 68000),
  ('Grace', 'IT', 59000);
```

## UPDATE

Modifies existing rows.

```sql
UPDATE employees
SET salary = 80000
WHERE id = 3;
```

- `SET` specifies which columns change and to what value
- `WHERE` restricts which rows are affected

### Multiple columns at once:
```sql
UPDATE employees
SET salary = 80000, dept = 'CS'
WHERE id = 3;
```

### THE DANGER: UPDATE without WHERE
```sql
UPDATE employees SET salary = 80000;
-- Changes EVERY employee's salary to 80000!
```

**Always double-check your WHERE clause before running UPDATE.**

### Updating based on a condition:
```sql
UPDATE orders SET status = 'shipped' WHERE customer_id = 10;
-- Affects ALL orders belonging to customer 10 (could be many rows)
```

## DELETE

Removes rows from a table.

```sql
DELETE FROM employees WHERE id = 3;
```

### THE DANGER: DELETE without WHERE
```sql
DELETE FROM employees;
-- Removes EVERY row. The table structure remains, but all data is gone.
```

**Always include a WHERE clause unless you intentionally want to delete all rows.**

## DELETE FROM vs DROP TABLE

| Statement | Effect |
|-----------|--------|
| `DELETE FROM employees;` | Removes all rows; table structure remains |
| `DELETE FROM employees WHERE id=3;` | Removes one row; table structure remains |
| `DROP TABLE employees;` | Removes the entire table including its definition |

`DELETE` is DML (Data Manipulation Language). `DROP TABLE` is DDL (Data Definition Language).

## Key Takeaways

- `INSERT INTO table (cols) VALUES (vals);` — add a row
- `UPDATE table SET col=val WHERE condition;` — modify rows
- `DELETE FROM table WHERE condition;` — remove rows
- **Never run UPDATE or DELETE without a WHERE clause unless you're sure**
- Multiple rows can be inserted in one `INSERT` by comma-separating value tuples
