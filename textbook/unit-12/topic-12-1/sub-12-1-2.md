# MySQL Command-Line Client

## Connecting to MySQL

```bash
mysql -u username -p
```

- `-u username` — specify the MySQL user
- `-p` — prompt for the password (do not type the password on the command line)
- Optional: `-h hostname` for a remote server, `-P port` for a non-default port

After connecting, you see the `mysql>` prompt.

## The Continuation Prompt (`->`)

If you press Enter without a semicolon, MySQL shows `->` and waits for more input. This means your statement is not yet terminated.

```
mysql> SELECT * FROM employees
    -> WHERE dept = 'CS'
    -> ;
```

Type `\c` to cancel a partially entered statement.

## Essential Commands

| Command | What it does |
|---------|-------------|
| `SHOW DATABASES;` | List all databases |
| `USE dbname;` | Switch to a database |
| `SHOW TABLES;` | List tables in current database |
| `DESCRIBE tablename;` | Show column definitions |
| `exit` or `quit` | Disconnect |

## Semicolon Rule

**Every SQL statement must end with a semicolon (;).** This is mandatory in the MySQL CLI (and most SQL clients). Forgetting it is the #1 source of the `->` continuation prompt.

## Example Session

```
$ mysql -u root -p
Enter password: ****
mysql> USE university;
Database changed
mysql> SHOW TABLES;
+----------------------+
| Tables_in_university |
+----------------------+
| courses              |
| enrollments          |
| students             |
+----------------------+
mysql> DESCRIBE students;
+-------+-------------+------+-----+---------+----------------+
| Field | Type        | Null | Key | Default | Extra          |
+-------+-------------+------+-----+---------+----------------+
| id    | int         | NO   | PRI | NULL    | auto_increment |
| name  | varchar(50) | YES  |     | NULL    |                |
| major | varchar(30) | YES  |     | NULL    |                |
+-------+-------------+------+-----+---------+----------------+
mysql> exit
Bye
```

## Key Takeaways

- Connect with `mysql -u user -p`
- End every statement with `;`
- `->` means MySQL is waiting for the semicolon
- `USE dbname;` selects which database to work with
- `SHOW DATABASES;` and `SHOW TABLES;` are your navigation tools
