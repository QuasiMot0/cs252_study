# SELECT / FROM / WHERE: SQL Query Basics

## The Basic SELECT Statement

```sql
SELECT column1, column2, ...
FROM tablename
WHERE condition;
```

- **SELECT** — which columns to return (projection)
- **FROM** — which table to query
- **WHERE** — which rows to include (filter)

All three clauses are part of a single query. The semicolon terminates it.

## Selecting All Columns

```sql
SELECT * FROM employees;
```

`*` means "all columns." Returns every column and every row (no filter).

## Selecting Specific Columns

```sql
SELECT name, salary FROM employees;
```

Returns only the `name` and `salary` columns for every row.

## Filtering with WHERE

```sql
SELECT name FROM employees WHERE dept = 'CS';
```

Returns the `name` column for rows where `dept` equals `'CS'`.

**Important:** SQL uses `=` (single equals) for equality, not `==`.

## Combining Conditions

```sql
SELECT name, salary FROM employees
WHERE dept = 'CS' AND salary > 60000;
```

- `AND` — both conditions must be true
- `OR` — either condition can be true
- `NOT` — negates a condition

## DISTINCT: Eliminating Duplicates

```sql
SELECT DISTINCT dept FROM employees;
```

Returns each department name only once, even if many employees share the same department.

## Case Insensitivity

SQL keywords (`SELECT`, `FROM`, `WHERE`) are **case-insensitive**. These are all equivalent:

```sql
SELECT * FROM employees WHERE dept = 'CS';
select * from employees where dept = 'CS';
Select * From employees Where dept = 'CS';
```

However, string comparisons in WHERE (`dept = 'CS'`) may be case-sensitive depending on the database's collation settings. In MySQL, the default collation is case-insensitive for strings.

## Common Mistakes

| Mistake | Problem |
|---------|---------|
| `SELECT name WHERE dept='CS'` | Missing `FROM` clause — syntax error |
| `WHERE salary == 50000` | `==` is not valid SQL; use `=` |
| `SELECT * WHERE dept='CS'` | Missing both `FROM` and table name |

## Key Takeaways

| Clause | Purpose |
|--------|---------|
| `SELECT col1, col2` | Choose which columns to return |
| `SELECT *` | Return all columns |
| `FROM table` | Specify which table |
| `WHERE condition` | Filter rows |
| `DISTINCT` | Remove duplicate rows |
