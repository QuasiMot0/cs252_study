# LIKE Operator: Pattern Matching with Wildcards

## The LIKE Operator

`LIKE` is used in a `WHERE` clause to match string patterns. It uses two wildcard characters:

| Wildcard | Matches |
|----------|---------|
| `%` | Zero or more characters (any characters) |
| `_` | Exactly one character (any character) |

```sql
SELECT * FROM students WHERE name LIKE 'pattern';
```

## The `%` Wildcard

Matches **zero or more** characters.

| Pattern | Matches | Does NOT match |
|---------|---------|---------------|
| `'A%'` | 'A', 'Alice', 'Aaron', 'ABC123' | 'alice' (case depends on collation) |
| `'%son'` | 'son', 'Mason', 'Jackson', 'Johnson' | 'sony' |
| `'%a%'` | Any string containing 'a' anywhere | Strings with no 'a' |
| `'%'` | Every string (matches anything) | — |

## The `_` Wildcard

Matches **exactly one** character.

| Pattern | Matches | Does NOT match |
|---------|---------|---------------|
| `'A_'` | 'Ab', 'A1', 'A!' (exactly 2 chars starting with A) | 'A', 'Alice' |
| `'___'` | Any 3-character string | 'ab', 'abcd' |
| `'J_n'` | 'Jan', 'Jon', 'J1n' (exactly 3 chars) | 'Jn', 'John' |

## Combining `%` and `_`

```sql
-- Names where position 2 and 3 are 'at', then anything:
WHERE name LIKE '_at%'
-- Matches: 'Batman', 'category', 'Patrick'
-- Does NOT match: 'at' (no char before 'at')

-- Names starting with J, any char, then 'n', then anything:
WHERE name LIKE 'J_n%'
-- Matches: 'Jan', 'Jon', 'Jan Smith', 'Jonathan'
```

## Practical Examples

```sql
-- Find all students whose name starts with 'A':
SELECT * FROM students WHERE name LIKE 'A%';

-- Find all emails from Purdue:
SELECT * FROM users WHERE email LIKE '%@purdue.edu';

-- Find all 5-digit zip codes:
SELECT * FROM addresses WHERE zip LIKE '_____';

-- Find products with 'pro' anywhere in the name:
SELECT * FROM products WHERE name LIKE '%pro%';
```

## Case Sensitivity

In MySQL with the default collation, `LIKE` is **case-insensitive**. `'A%'` matches 'alice', 'Alice', 'ALICE'. You can use `BINARY` for case-sensitive matching:

```sql
WHERE name LIKE BINARY 'A%'  -- case-sensitive
```

## Key Takeaways

- `%` = zero or more characters
- `_` = exactly one character
- Both can be combined: `'_at%'` = one char, then 'at', then anything
- LIKE is used for string pattern matching in WHERE
- MySQL LIKE is case-insensitive by default
