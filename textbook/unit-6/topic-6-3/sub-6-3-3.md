## The Three Standard Streams

From the slides:
> "Stdin (0), Stdout (1), Stderr (2) are file descriptors that a program inherits from its parent."
> "Usually they are the keyboard and the terminal window but they can be redirected."

| fd | Name | Default Source/Destination |
|---|---|---|
| 0 | stdin | keyboard input |
| 1 | stdout | terminal display |
| 2 | stderr | terminal display (error messages) |

These are **inherited from the parent process** (typically the shell) and can be redirected before execution begins.

## I/O Redirection Operators

From the slides:

| Operator | Effect |
|---|---|
| `command > file` | Redirect stdout to `file` (truncate) |
| `command >> file` | Redirect stdout to `file` (append) |
| `command < file` | Redirect stdin from `file` |
| `command 2> file` | Redirect stderr to `file` |
| `command > file 2>&1` | Redirect both stdout and stderr to `file` |

### Examples

```bash
ls -al > out.txt          # stdout goes to out.txt; stderr still goes to terminal
ls -al >> out.txt         # append stdout to out.txt
sort < names.txt          # sort reads from names.txt instead of keyboard
./prog 2> errors.txt      # stderr goes to errors.txt; stdout still to terminal
./prog > out.txt 2>&1     # both stdout and stderr go to out.txt
```

### 2>&1 Semantics

`2>&1` means "redirect fd 2 (stderr) to the same place fd 1 (stdout) is currently pointing." The order matters: `> out.txt 2>&1` first points fd 1 to `out.txt`, then points fd 2 to wherever fd 1 now points. Writing it as `2>&1 > out.txt` would point stderr to the old stdout (terminal), not the file.

## Pipes

From the slides:
> "Pipes connect the output of one command to the input of another."

```bash
ls -al | sort             # stdout of ls becomes stdin of sort
cat out.txt | grep ERROR  # stdout of cat becomes stdin of grep
ps -e | wc                # stdout of ps becomes stdin of wc
```

The shell creates a pipe, forks two child processes, and uses `dup2()` to wire the write-end to the left command's stdout and the read-end to the right command's stdin.

## Programmatic Redirection: dup() and dup2()

From the slides, the shell implements redirection using:

```c
dup2(fd1, fd2);   // make fd2 point to same Open File Object as fd1
dup(fd1);         // return a new fd pointing to same object as fd1
```

Typical shell redirect pattern before `exec()`:

```c
int fd = open("out.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
dup2(fd, 1);      // stdout now writes to out.txt
close(fd);        // clean up the extra fd
execvp(cmd, args);
```

## Key Terms

| Term | Meaning |
|---|---|
| stdin (fd 0) | Standard input; default = keyboard |
| stdout (fd 1) | Standard output; default = terminal |
| stderr (fd 2) | Standard error output; default = terminal |
| `>` | Redirect stdout to file (truncate) |
| `>>` | Redirect stdout to file (append) |
| `<` | Redirect stdin from file |
| `2>` | Redirect stderr to file |
| `2>&1` | Redirect stderr to same destination as stdout |
| Pipe (`\|`) | Connect stdout of left command to stdin of right command |
| `dup2(fd1, fd2)` | Redirect fd2 to point to same file object as fd1 |
