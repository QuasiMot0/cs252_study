## What Are Environment Variables?

From the slides:
> "Environment variables are a set of strings of the form A=B where A is the variable name and B is the value."
> "They are inherited by child processes."

Environment variables are key=value pairs attached to every process. They configure program behavior without requiring command-line arguments — things like where to find executables or who is logged in.

## The environ Array

From the slides:
> "`extern char **environ;`"
> "environ is an array of strings of the form A=B. The last element of the array is NULL."

```c
#include <stdio.h>

extern char **environ;

int main() {
    for (int i = 0; environ[i] != NULL; i++) {
        printf("%s\n", environ[i]);
    }
}
```

This is equivalent to running the `env` command.

## Common Environment Variables

From the slides, standard variables include:

| Variable | Meaning |
|---|---|
| `PATH` | Colon-separated list of directories searched for executables |
| `USER` | Login name of the current user |
| `HOME` | Home directory of the current user |

Example values:
```
PATH=/bin:/usr/bin:/usr/local/bin
USER=grr
HOME=/homes/grr
```

## Setting Environment Variables in the Shell

From the slides:
> "`export A=B` — A=B is set globally and all child processes will inherit A=B."
> "`A=B` — A=B is set locally and only works in the current shell."

```bash
export PATH=$PATH:/newdir    # add /newdir to end of PATH for this shell and all children
USER=grr                     # local to current shell only; children do not see it
```

`export` makes the variable visible to all programs launched from that shell. Without `export`, the variable is a shell-local variable that child processes cannot access.

## Viewing Environment Variables

```bash
env         # print all environment variables
echo $PATH  # print a specific variable
```

## Using getenv() in C

```c
#include <stdlib.h>

char *val = getenv("HOME");    // returns pointer to value string, or NULL if not set
```

## Key Terms

| Term | Meaning |
|---|---|
| Environment variable | A name=value string attached to a process; inherited by children |
| `extern char **environ` | C declaration for the array of environment strings (NULL-terminated) |
| `PATH` | Directories searched when running a command without a full path |
| `USER` | Current login name |
| `HOME` | Current user's home directory |
| `export A=B` | Set variable globally; all child processes inherit it |
| `A=B` (no export) | Set variable locally in the current shell only |
| `getenv("NAME")` | C function to retrieve an environment variable's value |
| `env` | Shell command to print all current environment variables |
