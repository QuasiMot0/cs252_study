## What Is a Process?

From the slides:
> "A process is a program in execution."

Every process has a set of **properties** that describe its identity and execution context.

## Process ID (PID)

From the slides:
> "The process ID (PID) is an index in the process table."
> "The PID uniquely identifies a process."
> "The init process has PID 0."
> "The OS will assign increasing PIDs."

```c
#include <unistd.h>

pid_t pid = getpid();    // returns this process's PID
```

## Command and Arguments (argc / argv)

From the slides:
> "`int main(int argc, char **argv)`"
> "argc — the number of arguments including the command."
> "argv[0] — the command, argv[1] — the first argument, etc."

```c
int main(int argc, char **argv) {
    // argc: count of arguments (including argv[0])
    // argv[0]: name of the program
    // argv[1]...argv[argc-1]: the actual arguments
}
```

Example: running `./hello world 42` gives `argc=3`, `argv[0]="./hello"`, `argv[1]="world"`, `argv[2]="42"`.

Printing all arguments:
```c
for (int i = 0; i < argc; i++) {
    printf("argv[%d] = %s\n", i, argv[i]);
}
```

## Current Working Directory

From the slides:
> "The current directory is the directory where the process is running."

Every process has a current working directory. Relative paths are resolved from this directory. It is inherited from the parent process (e.g., the shell).

## Process User ID (UID)

From the slides:
> "Every process has a user id (uid) which is the id of the user who started the process."
> "If the uid is 0, the process is the root process."

Special programs can change their effective UID:
- **setuid** bit on an executable — the process runs with the file owner's UID instead of the caller's
- **`su`** — substitute user; spawns a shell as another user
- **`sudo`** — run a single command as root

## Summary of Process Properties

| Property | How to Access / Where Set |
|---|---|
| PID | `getpid()`; assigned by OS at process creation |
| argc / argv | Parameters to `main()`; set by parent via `exec()` |
| Current directory | Inherited from parent; changed with `chdir()` |
| UID | Inherited from parent; root UID = 0 |
| Environment variables | Inherited from parent; accessible via `environ` or `getenv()` |
| Open file descriptors | Inherited from parent (fd 0/1/2 = stdin/stdout/stderr) |

## Key Terms

| Term | Meaning |
|---|---|
| PID | Process ID; unique integer index in the OS process table |
| `getpid()` | System call returning the calling process's PID |
| argc | Count of command-line arguments (includes argv[0]) |
| argv | Array of C strings: argv[0]=command, argv[1..argc-1]=arguments |
| Current directory | Directory from which relative paths are resolved; inherited from parent |
| UID | User ID of the process; 0 = root |
| setuid | File permission bit that makes a program run as the file's owner |
| `su` / `sudo` | Shell commands to run as another user or as root |
