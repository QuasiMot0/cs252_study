## Disk File System Structure

From the slides:
> "Disks when formatted are divided into sectors, tracks and blocks."
> "A Block is the intersection between a sector and a track."
> "Disks are logically divided into partitions. A partition is a group of blocks. Each partition is a different file system."

Each partition is divided into four regions:

| Region | Purpose |
|---|---|
| **Boot Block** | Code that jumps to the OS for loading |
| **Superblock** | Number of data blocks, number of inodes, bitmaps for used/free inodes and blocks, inode for root directory |
| **Inode List** | Array of inodes; one inode per file on the disk |
| **Data Blocks** | Actual file data |

## What Is an Inode?

From the slides:
> "An i-node represents a file in disk."

Each inode contains:

1. **Flag/Mode** — R/W/X permissions for Owner/Group/All; also whether it's a regular file, directory, device, or symbolic link
2. **Owners** — User ID (uid), Group ID (gid)
3. **Time Stamps** — Creation time, Access time, Modification time (stored in seconds since the Unix Epoch: 0:00, January 1, 1970)
4. **Size** — File size in bytes
5. **Reference Count** — Number of hard links (directories that reference this inode); file is deleted from disk when count reaches 0
6. **Block Index** — Pointers to the data blocks that make up the file

## The Block Index (Multi-Level)

From the slides:
> "To save space, the block index uses indices of different levels. This benefits small files since they form the largest percentage of files."

The inode contains:

| Pointer Type | Count | How It Works |
|---|---|---|
| **Direct** | 12 | Each points directly to a data block |
| **Single indirect** | 3 | Each points to a table of 256 block pointers |
| **Double indirect** | 1 | Points to a table of 256 → each points to a table of 256 block pointers |
| **Triple indirect** | 1 | Three levels of 256-entry tables |

## Block Size Math (1KB blocks, 256 pointers/block)

From the slides:

| Level | Capacity |
|---|---|
| Direct | 12 × 1 KB = **12 KB** |
| Single indirect | 3 × 256 × 1 KB = **768 KB** |
| Double indirect | 1 × 256 × 256 × 1 KB = **64 MB** |
| Triple indirect | 1 × 256 × 256 × 256 × 1 KB = **16 GB** |

For small files (most common), only direct blocks are needed: 1 disk access for the inode + 1 disk access for the data block.

## Alternative: Linked List

From the slides:
> "An alternative to the multi-level block index is a linked list. Every block will contain a pointer to the next block and so on. Linked lists are slow for random access."

The multi-level index is preferred because it allows O(1) random access to any block.

## Key Terms

| Term | Meaning |
|---|---|
| Block | Intersection of a track and sector on disk; unit of disk I/O |
| Partition | Logical group of blocks forming one file system |
| Superblock | Partition metadata: inode count, block count, bitmaps, root inode |
| Inode | Data structure representing one file; contains metadata and block pointers |
| Direct block | Inode pointer pointing directly to a data block; 12 in structure |
| Single indirect | Pointer to a block holding 256 block pointers |
| Double/Triple indirect | Two/three levels of indirection for very large files |
| Unix Epoch | January 1, 1970 00:00:00 UTC; timestamps stored as seconds since this |
