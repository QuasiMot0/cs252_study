## Where Permissions Are Stored

From the slides:
> "The permissions of a file in UNIX are stored in the inode in the flag bits."

Every inode contains a **flag/mode** field that encodes:
- File type (regular file, directory, device, symbolic link)
- Read/Write/Execute permissions for three groups

## The Three Permission Groups

From the slides, permissions are grouped as:

```
rwx  rwx  rwx
 ↓    ↓    ↓
User Group Other
```

Each triplet contains three bits:
- **r** (read) — view file contents / list directory
- **w** (write) — modify file / create or delete files in directory
- **x** (execute) — run as a program / traverse directory

## Viewing Permissions

```bash
ls -al
```

Example output:
```
-rw-rw-r--  1 grr  150 Aug 29 1995 calendar
-rw-------  1 grr  975 Mar 25 1999 cew.el
-rwxrwxr-x  1 grr 5924 Jul  9 10:48 chars
drwxr-sr-x 10 grr  512 Oct 14 1998 contools
```

The first character is `d` for directory, `-` for regular file.

## Changing Permissions: chmod (symbolic)

From the slides:
> "To change the permissions of a file use the command chmod."
> "`chmod <u|g|o><+|-><r|w|x> filename`"

- `u` = user (owner), `g` = group, `o` = others
- `+` = add, `-` = remove

Examples:
```bash
chmod u+rw hello.txt     # user can read and write
chmod g+rw hello.txt     # group can read and write
chmod o+r  hello.txt     # others can read
chmod o-w  hello.txt     # remove write from others
chmod ugo+x myscript.sh  # everyone can execute (required for scripts)
```

From the slides:
> "Scripts and Executable files should have the executable bit set to be able to execute them."

## Changing Permissions: chmod (octal)

From the slides:
> "Also you can change the permission bits all at once using the bit representation in octal."

Each rwx group is a 3-bit binary number:

```
USER  GROUP  OTHERS
 RWX    RWX     RWX
 110    110     100   ← binary
   6      6       4   ← octal digit

chmod 664 hello.c
```

| Octal | Binary | Permissions |
|---|---|---|
| 7 | 111 | rwx |
| 6 | 110 | rw- |
| 5 | 101 | r-x |
| 4 | 100 | r-- |
| 0 | 000 | --- |

Common octal values:
- `chmod 755 file` → rwxr-xr-x (owner full, group/others read+exec)
- `chmod 644 file` → rw-r--r-- (owner rw, group/others read only)
- `chmod 664 file` → rw-rw-r-- (owner+group rw, others read only)

## Key Terms

| Term | Meaning |
|---|---|
| Flag/Mode field | Inode field storing file type and permission bits |
| rwx | Read, Write, Execute permission bits |
| User/Group/Other | The three permission groups; 9 bits total |
| `ls -al` | Show files with permissions, owner, size, date |
| `chmod u+x file` | Symbolic chmod: add execute permission for owner |
| `chmod 664 file` | Octal chmod: set all permissions at once |
| Executable bit | Required for scripts and binaries to be runnable |
