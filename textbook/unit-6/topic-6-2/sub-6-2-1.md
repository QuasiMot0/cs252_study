## What Is a File Descriptor?

A **file descriptor (fd)** is a small non-negative integer that a process uses to refer to an open file. Internally, it is an **index into the process's per-process file descriptor table**.

From the slides:
> "A file descriptor is an integer that refers to an open file."
> "The maximum number of file descriptors a process can have is 1024."

## The Three-Level Structure

```
Per-Process Table          System Open File Table        Inode Table
┌──────────────┐          ┌───────────────────────┐    ┌──────────────┐
│ fd 0 (stdin) │ ──────▶  │ I-Node (major, minor) │ ▶  │  Inode data  │
│ fd 1 (stdout)│ ──────▶  │ Open Mode (R/W/Append)│    └──────────────┘
│ fd 2 (stderr)│ ──────▶  │ Offset (current pos)  │
│ fd 3         │ ──────▶  │ Reference Count       │
│  ...         │          └───────────────────────┘
│ fd 1023      │
└──────────────┘
```

Each entry in the per-process table points to a **system Open File Object** which contains:

| Field | Description |
|---|---|
| **I-Node** | Major and minor device + inode number identifying the file on disk |
| **Open Mode** | How the file was opened: Read, Write, or Append |
| **Offset** | Current position in the file (advances on each read/write) |
| **Reference Count** | Number of fd table entries (across all processes) pointing to this object |

## Default File Descriptors

Every process starts with three file descriptors already open:

| fd | Name | Default |
|---|---|---|
| 0 | stdin | keyboard |
| 1 | stdout | terminal |
| 2 | stderr | terminal |

These are **inherited from the parent process** when a new process is created.

## Reference Count Behavior

When a process is `fork()`ed, the child gets a copy of the parent's fd table. Both entries point to the **same Open File Object**, so the reference count is incremented. The file object is not freed until the reference count reaches 0 — i.e., all processes that share it have closed their copy.

## Key Terms

| Term | Meaning |
|---|---|
| File descriptor (fd) | Integer index into the per-process fd table |
| Per-process fd table | Maps fd integers to Open File Objects; max 1024 entries |
| Open File Object | System-level structure: inode, mode, offset, ref count |
| Offset | Current read/write position; shared when fd is inherited via fork |
| Reference count | Number of fd table entries pointing to this Open File Object |
| fd 0 / 1 / 2 | stdin / stdout / stderr; inherited from parent |
