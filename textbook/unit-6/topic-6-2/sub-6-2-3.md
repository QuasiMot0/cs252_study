## fd Inheritance Across fork()

When a process calls `fork()`, the child receives a **copy of the parent's per-process fd table**. Each fd in the child points to the **same Open File Object** in the system open file table.

Because both parent and child point to the same object:
- The **offset** is shared вҖ” if the parent advances the read position, the child sees the new position.
- The **reference count** of each shared Open File Object is incremented by 1 for each new fd pointing to it.

## dup() and dup2()

These system calls create additional fd entries pointing to the same Open File Object.

```c
int newfd = dup(oldfd);
```
Returns the lowest available fd number; `newfd` points to the same file object as `oldfd`.

```c
dup2(oldfd, newfd);
```
Makes `newfd` point to the same file object as `oldfd`. If `newfd` was already open, it is closed first. Used to redirect stdin/stdout/stderr before `exec()`.

### Redirect stdout to a file (shell pipe implementation)

```c
int fd = open("out.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
dup2(fd, 1);   // fd 1 (stdout) now points to out.txt
close(fd);     // no longer need the original fd
```

After `dup2(fd, 1)`, anything written to `stdout` (fd 1) goes to `out.txt`.

## Shared Offset Behavior

Because the offset lives in the Open File Object (not the fd table entry), parent and child **share the same offset**. If the parent reads 100 bytes, the child's next `read()` on the same fd continues from byte 100, not byte 0.

This is the key mechanism the shell uses to implement pipes and redirections: `fork()` inherits the open file, then `dup2()` rewires the fd before `exec()`.

## Key Terms

| Term | Meaning |
|---|---|
| fd inheritance | Child process gets a copy of parent's fd table on `fork()` |
| Shared offset | Parent and child share the same file position via the Open File Object |
| Reference count | Incremented for each fd pointing to an Open File Object; object freed at 0 |
| `dup(fd)` | Returns new fd pointing to same file object as `fd` |
| `dup2(oldfd, newfd)` | Redirects `newfd` to point to same object as `oldfd`; closes `newfd` first |
| Shell redirect | Uses `fork()` + `dup2()` + `exec()` to wire up pipes and file redirections |
