## Redirection Operators

The shell grammar defines `io_modifier` rules that recognize these operators:

| Syntax | Token(s) | Effect |
|---|---|---|
| `> file` | `GREAT WORD` | Redirect stdout to file (truncate/create) |
| `>> file` | `GREATGREAT WORD` | Redirect stdout to file (append) |
| `< file` | `LESS WORD` | Redirect stdin from file |
| `2> file` | `TWOGREAT WORD` | Redirect stderr (fd 2) to file |
| `>& file` | `GREATAMPERSAND WORD` | Redirect both stdout and stderr to file |

## How Each Is Implemented

### `> file` ŌĆö stdout truncate

```c
int fd = open(file, O_WRONLY | O_CREAT | O_TRUNC, 0644);
dup2(fd, 1);    // stdout ŌåÆ file
close(fd);
```

The `O_TRUNC` flag clears the file if it already exists.

### `>> file` ŌĆö stdout append

```c
int fd = open(file, O_WRONLY | O_CREAT | O_APPEND, 0644);
dup2(fd, 1);    // stdout ŌåÆ file (writes go to end)
close(fd);
```

The `O_APPEND` flag makes every `write()` go to the current end of the file.

### `< file` ŌĆö stdin from file

```c
int fd = open(file, O_RDONLY);
dup2(fd, 0);    // stdin ŌåÉ file
close(fd);
```

### `2> file` ŌĆö stderr redirect

```c
int fd = open(file, O_WRONLY | O_CREAT | O_TRUNC, 0644);
dup2(fd, 2);    // stderr (fd 2) ŌåÆ file
close(fd);
```

### `>& file` ŌĆö stdout and stderr to same file

Open the file once, then `dup2` to both fd 1 and fd 2:

```c
int fd = open(file, O_WRONLY | O_CREAT | O_TRUNC, 0644);
dup2(fd, 1);    // stdout ŌåÆ file
dup2(fd, 2);    // stderr ŌåÆ file
close(fd);
```

## The >& Pointer Aliasing Bug

**Exam question (lab3_quiz Q15):** In the Yacc action for `>& file`, the grammar matched the file name as `$2` (a `std::string*`). The naive code:

```cpp
_errFile = $2;      // BUG: both point to the same std::string object
_outFile = $2;
```

Both `_errFile` and `_outFile` end up pointing to the same `std::string` object. If one is freed or modified, the other is corrupted.

**Fix:**

```cpp
_errFile = new std::string(*$2);    // make an independent copy
_outFile = $2;
```

Now each field owns a separate string.

## Debugging Note

When debugging your shell with `fprintf(stderr, "debug\n")`, do **not** redirect stderr until stdout/stdin redirections are working. Otherwise your debug messages disappear into the redirect target and you cannot see them.

## Key Terms

| Term | Meaning |
|---|---|
| `O_TRUNC` | Open flag: truncate file to zero length if it exists |
| `O_APPEND` | Open flag: all writes go to current end of file |
| `O_CREAT` | Open flag: create file if it does not exist |
| Pointer aliasing | Two pointers pointing to the same object; changes through one affect the other |
