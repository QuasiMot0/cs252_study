## dup() for Saving Descriptors

Before the shell manipulates file descriptors for a pipeline, it saves the current stdin and stdout using `dup()`:

```c
int tmpin  = dup(0);   // save stdin  → tmpin  points to same file as fd 0
int tmpout = dup(1);   // save stdout → tmpout points to same file as fd 1
```

After all commands have been forked and run, the executor restores them:

```c
dup2(tmpin,  0);   close(tmpin);
dup2(tmpout, 1);   close(tmpout);
```

This lets the shell keep using its own terminal stdin/stdout between commands.

## Why Redirection Order Matters

`dup2` is a **snapshot at the time of the call**. It copies the current target of the source fd. This means the order in which you apply `dup2` calls determines the final mapping.

### Case 1: `> file 2>&1`

Intention: send stdout to `file` and stderr to the same place as stdout (i.e., `file`).

```c
// 1. Redirect stdout to file
int fd = open("file", O_WRONLY|O_CREAT|O_TRUNC, 0644);
dup2(fd, 1);    // fd 1 → file
close(fd);

// 2. Redirect stderr to where stdout currently points (file)
dup2(1, 2);     // fd 2 → file (same file object as fd 1)
```

Result: both stdout and stderr go to `file`. ✓

### Case 2: `2>&1 > file`

Same tokens, reversed order.

```c
// 1. Redirect stderr to where stdout currently points (terminal)
dup2(1, 2);     // fd 2 → terminal (stdout hasn't been redirected yet!)

// 2. Redirect stdout to file
int fd = open("file", O_WRONLY|O_CREAT|O_TRUNC, 0644);
dup2(fd, 1);    // fd 1 → file
close(fd);
```

Result: stdout goes to `file`, but stderr still goes to the **terminal**. ✗

**The lesson:** `2>&1` must come *after* `>file` if you want both to go to the same file.

## When Order Doesn't Matter

**Exam question (lab3_quiz Q9):** If both `dup2` calls use the **same source fd**, order does not matter. For example, if you are redirecting stdin and stdout from/to two different already-open file descriptors, the calls are independent and can happen in either order. Order only matters when one redirection depends on the current state set by a previous one.

## Key Terms

| Term | Meaning |
|---|---|
| `dup(fd)` | Create a new fd pointing to the same file object as `fd` |
| `tmpin` / `tmpout` | Shell-internal saved copies of original stdin/stdout before pipeline setup |
| Snapshot semantics | `dup2(src, dst)` copies what `src` points to *right now*, not dynamically |
