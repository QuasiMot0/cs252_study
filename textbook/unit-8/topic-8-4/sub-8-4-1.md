## The Open File Table

Every process has a **file descriptor table** — a small array of integers. Each index (file descriptor, or fd) is a slot pointing to an entry in the kernel's open file table. The kernel's table tracks the actual file, its current position, and a **reference count** (how many fds from how many processes point to it).

Standard file descriptors assigned at startup:

| fd | Name | Purpose |
|---|---|---|
| 0 | stdin | standard input |
| 1 | stdout | standard output |
| 2 | stderr | standard error |

## dup2(oldfd, newfd)

```c
int dup2(int oldfd, int newfd);
```

`dup2` makes `newfd` point to the same open file object as `oldfd`:

1. If `newfd` is currently open, it is first closed (its reference count is decremented).
2. `newfd` is then made to point to the same open file object as `oldfd`.
3. The reference count of that open file object is incremented.
4. `oldfd` is unchanged.

After `dup2(oldfd, newfd)`, both `oldfd` and `newfd` refer to the same underlying file.

## Redirecting stdout to a File

To make a child process write to `output.txt` instead of the terminal:

```c
// In the child, before execvp:
int fd = open("output.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
if (fd == -1) { perror("open"); _exit(1); }

dup2(fd, 1);    // make stdout (fd 1) point to output.txt's file object
close(fd);      // fd is redundant now; close it (the file stays open via fd 1)

execvp(args[0], args);
```

Before `dup2`:
```
fd 1 (stdout) → terminal
fd 5          → output.txt
```

After `dup2(5, 1)`:
```
fd 1 (stdout) → output.txt  ← now points here
fd 5          → output.txt  ← redundant; close it
```

After `close(5)`:
```
fd 1 (stdout) → output.txt  ← still open; only reference count dropped for fd 5
```

**Exam question (lab3_quiz Q5):** After `dup2(fd, 1)` and `close(fd)`, `printf` (which writes to fd 1) goes to the file. Closing the original `fd` does not affect the redirect because fd 1 still holds its own reference to the file object.

## Redirecting stdin from a File

To make stdin read from a file instead of the keyboard:

```c
int fd = open("input.txt", O_RDONLY);
dup2(fd, 0);    // make stdin (fd 0) read from input.txt
close(fd);
```

## Saving and Restoring with dup()

The shell itself needs to use stdin/stdout to interact with the user. Before setting up a pipeline, the executor saves the original stdin/stdout using `dup()` (which creates a new fd pointing to the same file object):

```c
int tmpin  = dup(0);   // save current stdin
int tmpout = dup(1);   // save current stdout
// ... set up redirections, fork, exec ...
dup2(tmpin,  0);       // restore stdin
dup2(tmpout, 1);       // restore stdout
close(tmpin);
close(tmpout);
```

## Key Terms

| Term | Meaning |
|---|---|
| File descriptor (fd) | Integer index into the process's file descriptor table |
| Open file table | Kernel structure tracking open files, positions, and reference counts |
| `dup2(oldfd, newfd)` | Make `newfd` point to the same file object as `oldfd` |
| `dup(fd)` | Create a new fd pointing to the same file object as `fd` |
| Reference count | Number of fds (across all processes) pointing to a file object |
