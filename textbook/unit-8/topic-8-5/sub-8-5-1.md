## What pipe() Does

```c
int pipe(int fd[2]);
```

`pipe()` creates a unidirectional data channel (a pipe) in the kernel and gives you two file descriptors to access its two ends:

| fd | End | Direction |
|---|---|---|
| `fd[0]` | read end | data flows **out** of the pipe here |
| `fd[1]` | write end | data flows **into** the pipe here |

Data written to `fd[1]` can be read from `fd[0]`. On Linux, pipes are unidirectional — data only flows in one direction.

```
         write          read
Program ──────► fd[1] ──► fd[0] ──────► Program
                  ↑ pipe ↑
```

## Closing Unused Ends

After `fork()`, both parent and child have copies of both file descriptors. Each process should **close the ends it does not use**. This is critical for correct EOF behavior.

**Why it matters:** A `read()` on `fd[0]` blocks until:
- Data is available, OR
- **All** file descriptors referring to the write end (`fd[1]`) are closed

If even one process holds `fd[1]` open without writing, `read()` on `fd[0]` will block forever waiting for data that will never come.

## Correct Pipe Usage Pattern

```c
int fdpipe[2];
pipe(fdpipe);           // fdpipe[0] = read, fdpipe[1] = write

pid_t ret = fork();
if (ret == 0) {
    // Child writes to the pipe
    close(fdpipe[0]);              // child doesn't read → close read end
    dup2(fdpipe[1], 1);            // stdout → pipe write end
    close(fdpipe[1]);              // redundant now; close it
    execvp(args[0], args);
    _exit(1);
} else {
    // Parent reads from the pipe
    close(fdpipe[1]);              // parent doesn't write → close write end
    // ... read from fdpipe[0] or pass it to next command ...
}
```

**Exam question (lab3_quiz Q3):** When both ends are closed correctly, the reader gets EOF when the writer is done and exits. The pipe produces the expected output.

**Exam question (lab3_quiz Q4):** If the parent forgets to close `fdpipe[1]`, the reader's `read()` will block forever even after the child exits, because the parent still holds the write end open. This is a deadlock.

## Key Terms

| Term | Meaning |
|---|---|
| `pipe(fd[2])` | Create a unidirectional pipe; `fd[0]`=read, `fd[1]`=write |
| Unidirectional | Data flows in one direction only |
| EOF | Signaled to the reader when all write-end references are closed |
| Deadlock | A state where a process blocks forever waiting for a condition that will never be satisfied |
