## Signal Types Relevant to the Shell

| Signal | Cause | Default action |
|---|---|---|
| `SIGINT` | User presses Ctrl-C | Terminate the process |
| `SIGCHLD` | A child process exits | Ignored (on most systems) |
| `SIGKILL` | `kill -9 pid` | Terminate; **cannot be caught or ignored** |
| `SIGSEGV` | Invalid memory access | Terminate with core dump |

## SIGINT and the Shell

When the user presses Ctrl-C, the kernel sends `SIGINT` to the foreground process group. A shell typically installs a custom handler for `SIGINT` so it can print a new prompt line rather than killing the entire shell.

**Exam question (lab3_quiz Q6):** If a custom `SIGINT` handler simply prints a message and returns, the shell **continues running** â€” the default terminate behavior is replaced by the handler's behavior.

## Signals and the Lex Scanner

The Flex-generated scanner calls `getc()` internally to read the next character from stdin. `getc()` calls the `read()` system call. System calls that block (like `read()`) are **interruptible**: if a signal arrives while `read()` is blocked, `read()` returns `-1` and sets `errno` to `EINTR`.

This means:
- Ctrl-C (`SIGINT`) interrupts the scanner's `read()`.
- A child exiting (`SIGCHLD`) also interrupts the scanner's `read()`.
- On interruption, `getc()` returns -1/EOF and the scanner exits â€” not what you want.

## Using SA_RESTART

To prevent signals from interrupting blocking system calls, install signal handlers with `sigaction()` and set the `SA_RESTART` flag. With `SA_RESTART`, the kernel automatically retries the interrupted system call after the signal handler returns.

```c
#include <signal.h>

struct sigaction signalAction;
signalAction.sa_handler = sigIntHandler;   // custom handler function
sigemptyset(&signalAction.sa_mask);        // no signals blocked during handler
signalAction.sa_flags = SA_RESTART;        // restart interrupted syscalls

int error = sigaction(SIGINT, &signalAction, NULL);
if (error) {
    perror("sigaction");
    exit(-1);
}
```

Call `sigaction` **once per signal** in `main()`. With `SA_RESTART`, when Ctrl-C arrives while the scanner is blocked in `read()`, the handler runs and then `read()` resumes automatically.

## Why sigaction() Instead of signal()

`signal()` is the older, simpler API, but its behavior is not portable â€” on some systems a signal handler is reset to default after the first invocation. `sigaction()` has well-defined, portable semantics and supports flags like `SA_RESTART`. The CS252 shell uses `sigaction()`.

## Key Terms

| Term | Meaning |
|---|---|
| `SIGINT` | Signal sent when user presses Ctrl-C |
| `SIGCHLD` | Signal sent to parent when any child exits |
| `EINTR` | `errno` value set when a syscall is interrupted by a signal |
| `SA_RESTART` | Flag telling the kernel to automatically restart interrupted blocking syscalls |
| `sigaction()` | POSIX system call to install a signal handler with precise control over flags |
| Signal handler | A function that the kernel calls when the process receives a signal |
