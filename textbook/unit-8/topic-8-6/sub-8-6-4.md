## What $? Means

In a UNIX shell, `$?` expands to the exit status of the most recently executed foreground command. A value of `0` conventionally means success; any non-zero value means failure.

```bash
ls /tmp       # exits with 0 (success)
echo $?       # prints: 0

ls /nonexistent    # exits with 2 (error)
echo $?            # prints: 2
```

## How the Shell Collects the Exit Status

After calling `waitpid` for a foreground child, the shell uses the `WEXITSTATUS` macro to extract the child's exit code from the raw `wstatus` value:

```c
int wstatus;
waitpid(child_pid, &wstatus, 0);

if (WIFEXITED(wstatus)) {
    int exit_code = WEXITSTATUS(wstatus);
    // Store exit_code for $? expansion
}
```

- `WIFEXITED(wstatus)` — true if the child exited normally (called `exit()` or returned from `main`)
- `WEXITSTATUS(wstatus)` — extracts the 8-bit exit code from the `wstatus` integer

The shell stores this value (e.g., in a global variable) so that when the user subsequently types `$?` or uses it in a condition, the lexer can substitute it.

## Where $? Expansion Happens

`$?` is recognized at lex time (in `shell.l`). When the scanner sees `$?`, it calls `getenv` or reads the stored exit status variable and uses `myunputc()` to inject the decimal string representation into the lex input buffer. The parser then sees the digits as a `WORD` token.

The same expansion point that handles `$VAR` (environment variable substitution) handles `$?` as a special case.

## Exit Status of Pipelines

In most shells, the exit status of a pipeline `cmd1 | cmd2 | cmd3` is the exit status of the **last command** (`cmd3`). The shell must therefore keep track of which child is the last in the pipeline so it can store that child's exit code as `$?`.

## Key Terms

| Term | Meaning |
|---|---|
| `$?` | Shell special variable: exit status of the last foreground command |
| `WIFEXITED(s)` | Macro: true if the child exited normally |
| `WEXITSTATUS(s)` | Macro: extracts the 8-bit exit code from `wstatus` |
| `wstatus` | Raw integer filled by `waitpid`; encodes how and why the child terminated |
