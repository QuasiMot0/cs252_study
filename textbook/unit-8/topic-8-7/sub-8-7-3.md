## What Subshell Expansion Does

The syntax `` `command` `` or `$(command)` runs `command` in a child shell process and substitutes its standard output back into the current command line. For example:

```bash
export DAYOFWEEK=$(date | awk '{print $1;}')
echo $(cat a.out)
```

In the second example, the shell runs `cat a.out`, captures its output, and injects that text as arguments to `echo`.

## Adding Subshell Syntax to shell.l

To recognize `` `…` `` and `$(…)`, new Flex rules must be added to `shell.l` to detect the opening delimiter and then collect the command text inside.

## Implementation: Two-Pipe Protocol

The shell implements subshell expansion with two pipes:

```
          pin[1] ──► pin[0]
Shell ─────────────────────► Subshell
  ▲    pout[0] ◄── pout[1]     │
  └────────────────────────────┘
```

### Step-by-step

1. **Parent creates two pipes:**
   ```c
   int pin[2], pout[2];
   pipe(pin);    // pin[1] = write-to-subshell, pin[0] = subshell's stdin
   pipe(pout);   // pout[1] = subshell writes here, pout[0] = parent reads output
   ```

2. **Parent forks a child.**

3. **In the child:**
   - Redirect stdin from `pin[0]` using `dup2(pin[0], 0)`
   - Redirect stdout to `pout[1]` using `dup2(pout[1], 1)`
   - Close unused ends: `close(pin[1])` and `close(pout[0])`
   - Execute the shell itself: `execvp("/proc/self/exe", args)` — the child is a new instance of the shell

4. **In the parent:**
   - Close ends the parent does not use: `close(pin[0])` and `close(pout[1])`

5. **Parent sends the command:**
   ```c
   write(pin[1], "cat a.out\n", strlen("cat a.out\n"));
   write(pin[1], "exit\n",      5);
   close(pin[1]);   // signal EOF to child (so child's shell exits)
   ```

6. **Child (subshell) runs the command** and writes output to `pout[1]`.

7. **Parent reads the output:**
   - Close `pout[1]` (if not done yet) — allows EOF to propagate
   - Read all output from `pout[0]` into a buffer

8. **Parent injects the buffer into the lex input stream** using `myunputc()`.

## Injecting into the Lex Buffer with myunputc()

The output from the subshell must appear to the lexer as if the user typed it. `myunputc()` is defined at the top of `shell.l` and pushes one character into Flex's internal input buffer.

**Critical detail:** Flex's input buffer is **LIFO** — the last character pushed is the first one read. To inject the string `"hello"`, you must push the characters in **reverse order**:

```c
// Inject "hello" — push from last char to first
for (int i = strlen(buf) - 1; i >= 0; i--) {
    myunputc(buf[i]);
}
```

**Exam question (lab3_quiz Q13):** `myunputc` must be called in reverse order (last character first) because the lex buffer is LIFO.

**Exam question (lab3_quiz Q18):** `$()` injects the text of the command's output directly into the lex input stream. (By contrast, `<()` creates a named pipe (FIFO) and passes the FIFO's filename as the argument — a different mechanism.)

## Key Terms

| Term | Meaning |
|---|---|
| Subshell | A child process that runs its own instance of the shell |
| `pin` | Pipe used to send the command text **to** the subshell's stdin |
| `pout` | Pipe used to receive the subshell's output **from** its stdout |
| `/proc/self/exe` | Linux path to the current process's own executable — used to exec a new shell instance |
| `myunputc(c)` | Push character `c` into Flex's LIFO input buffer |
| LIFO | Last In, First Out — the order characters must be pushed to inject them in reading order |
