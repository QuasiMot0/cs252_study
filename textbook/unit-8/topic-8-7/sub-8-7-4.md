## Environment Variable API

The C standard library provides three functions for working with environment variables:

```c
char *getenv(const char *name);
// Returns the value of the env var 'name', or NULL if not set.

int setenv(const char *name, const char *value, int overwrite);
// Sets 'name' to 'value'. If 'overwrite' is non-zero, replaces
// existing value. Returns 0 on success, -1 on error.

int unsetenv(const char *name);
// Removes 'name' from the environment. Returns 0 on success.
```

These are the same functions used by the `setenv` and `unsetenv` built-in commands in the shell.

## $VAR Expansion in shell.l

When the Flex scanner encounters `$VARNAME` (or `${VARNAME}`), it performs the expansion at lex time, before the token is passed to Yacc:

1. Extract the variable name from `yytext` (everything after `$`).
2. Call `getenv(name)` to get its value.
3. If the value is non-NULL, inject it into the lex input buffer using `myunputc()` in reverse order so it reads correctly (the buffer is LIFO).
4. If the variable is not set, inject an empty string (nothing).

The Yacc parser never sees `$VARNAME` — it sees only the expanded value as one or more `WORD` tokens.

## $? Expansion

`$?` is a special case handled the same way: the shell stores the last command's exit code in a variable, and when `$?` is seen in `shell.l`, the decimal string of that exit code is injected via `myunputc()`.

## Tilde Expansion

A leading `~` in an argument expands to the user's home directory. This is also done at lex time:

1. When the scanner sees a word beginning with `~`, extract any path component after `~`.
2. Call `getenv("HOME")` to get the home directory path.
3. Inject the home directory string followed by any remaining path using `myunputc()` in reverse order.

For example, `~/documents` expands to `/home/user/documents` (where `/home/user` is `$HOME`).

## Why Expansion Happens at Lex Time

Performing expansions in the Flex scanner (lex time) is simpler than trying to handle them in the executor:

- The scanner already has direct access to `myunputc()` to inject text into the input stream.
- Expanding before Yacc sees the token means the grammar rules do not need to know about `$VAR` or `~` syntax at all — they just see normal `WORD` tokens.
- This matches how real shells work: variable and tilde expansion happen as part of word processing before the command is dispatched.

## printenv: Viewing All Variables

The `printenv` command (a builtin in the CS252 shell that runs in a child process) prints all environment variables. The `environ` global variable is a `char**` that the OS provides — a NULL-terminated array of strings of the form `NAME=value`:

```c
extern char **environ;
char **p = environ;
while (*p != NULL) {
    printf("%s\n", *p);
    p++;
}
```

## Key Terms

| Term | Meaning |
|---|---|
| `getenv(name)` | Get the value of an environment variable |
| `setenv(name, val, overwrite)` | Set an environment variable |
| `unsetenv(name)` | Remove an environment variable |
| `environ` | Global `char**` array of all environment variables |
| `$VAR` | Shell syntax to substitute the value of env var VAR |
| `~` | Shell syntax to substitute the user's home directory (`$HOME`) |
| `myunputc()` | Injects a character into Flex's LIFO input buffer |
