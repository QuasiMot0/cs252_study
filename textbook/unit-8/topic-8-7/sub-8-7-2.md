## The Problem with Simple Wildcard Expansion

The flat expansion algorithm (using `opendir(".")` + `regexec`) only works for wildcards in a single directory level. It cannot handle patterns like `/p*/*a/b*/aa*` where wildcards appear in multiple path components separated by `/`.

## The prefix/suffix Strategy

The key insight is to expand one path **component** at a time, left to right. The function `expandWildcard(prefix, suffix)` is used:

- **`prefix`** — the part of the path that has already been expanded. It contains no wildcards.
- **`suffix`** — the remaining part of the path that still needs to be expanded. It may contain wildcards.

```
/usr/l*/a*
 ↑prefix  ↑suffix
```

`expandWildcard` is initially called with an empty prefix and the full wildcard pattern as the suffix:

```c
expandWildcard("", "/usr/l*/a*");
```

## Base Case: Suffix Is Empty

When `suffix[0] == 0` (the suffix is empty), the prefix is fully expanded — insert it as an argument:

```c
if (suffix[0] == 0) {
    Command::_currentSimpleCommand->insertArgument(strdup(prefix));
    return;
}
```

## Recursive Case: Extract Next Component

Split the suffix at the first `/` to get the next path component:

```c
char *s = strchr(suffix, '/');
char component[MAXFILENAME];

if (s != NULL) {
    // Copy up to the first '/'
    strncpy(component, suffix, s - suffix);
    component[s - suffix] = '\0';
    suffix = s + 1;   // advance suffix past the '/'
} else {
    // Last component — copy entire suffix
    strcpy(component, suffix);
    suffix = suffix + strlen(suffix);   // suffix becomes empty
}
```

## Two Sub-cases for the Component

### Component has no wildcards

Append the component to the prefix and recurse:

```c
if (/* component has neither '*' nor '?' */) {
    char newPrefix[MAXFILENAME];
    sprintf(newPrefix, "%s/%s", prefix, component);
    expandWildcard(newPrefix, suffix);
    return;
}
```

### Component has wildcards

Compile the component to a regex, open the directory named by `prefix` (or `"."` if prefix is empty), and test each entry:

```c
char *expbuf = compile(component, /* flags */);
char *dir = (prefix is empty) ? "." : prefix;
DIR *d = opendir(dir);
if (d == NULL) return;

struct dirent *ent;
while ((ent = readdir(d)) != NULL) {
    if (advance(ent->d_name, expbuf)) {
        // Entry matches — recurse with extended prefix
        char newPrefix[MAXFILENAME];
        sprintf(newPrefix, "%s/%s", prefix, ent->d_name);
        expandWildcard(newPrefix, suffix);
    }
}
close(d);
```

Each matching directory entry becomes part of the new prefix, and `expandWildcard` recurses with the remaining suffix.

## Example: Expanding `/u*/l*/a*`

```
Call: expandWildcard("", "/u*/l*/a*")
  Component = "u*", suffix = "l*/a*"
  → matches /usr, /u, /unix in root
  
  Call: expandWildcard("/usr", "l*/a*")
    Component = "l*", suffix = "a*"
    → matches /usr/lib, /usr/local
    
    Call: expandWildcard("/usr/lib", "a*")
      Component = "a*", suffix = ""
      → matches /usr/lib/aa, /usr/lib/abb
      
      Call: expandWildcard("/usr/lib/aa", "")
        suffix is empty → insert "/usr/lib/aa"
      Call: expandWildcard("/usr/lib/abb", "")
        suffix is empty → insert "/usr/lib/abb"
    
    Call: expandWildcard("/usr/local", "a*") ...
  
  Call: expandWildcard("/unix", "l*/a*") ...
```

## Key Terms

| Term | Meaning |
|---|---|
| `prefix` | Part of the path already fully expanded (no wildcards) |
| `suffix` | Remaining path to expand (may contain wildcards) |
| Component | One segment of the path between `/` separators |
| `expandWildcard(prefix, suffix)` | Recursive function that expands one component at a time |
| Base case | When suffix is empty: insert prefix as a final argument |
