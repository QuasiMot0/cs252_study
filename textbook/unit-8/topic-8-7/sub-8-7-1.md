## What Wildcard Expansion Does

When the user types `*.c`, the shell does not pass the literal string `*.c` to the program. Instead it **expands** the wildcard into a sorted list of filenames in the current directory that match the pattern, then inserts each filename as a separate argument. This expansion happens in `shell.y` (the Yacc file) where arguments are inserted into the current command.

## Entry Point: expandWildcardsIfNecessary

```c
void expandWildcardsIfNecessary(char *arg)
{
    // Return early if no wildcard characters present
    if (strchr(arg, '*') == NULL && strchr(arg, '?') == NULL) {
        Command::_currentSimpleCommand->insertArgument(arg);
        return;
    }
    // ... do expansion ...
}
```

If `arg` contains neither `*` nor `?`, it is inserted directly as an argument. Otherwise, the full expansion algorithm runs.

## Step 1: Convert Wildcard to Regular Expression

The wildcard pattern must be converted to a POSIX regular expression so that `regexec()` can test filenames against it:

| Wildcard char | Regex equivalent |
|---|---|
| `*` | `.*` (zero or more of any character) |
| `?` | `.` (exactly one of any character) |
| `.` | `\.` (literal dot — must be escaped) |

The result is anchored with `^` at the start and `$` at the end so the regex matches the entire filename, not just a substring.

```c
// Allocate enough space: each char may expand to 2 chars, plus "^…$\0"
char *reg = (char*)malloc(2 * strlen(arg) + 10);
char *a = arg;
char *r = reg;
*r = '^'; r++;
while (*a) {
    if      (*a == '*') { *r = '.'; r++; *r = '*'; r++; }
    else if (*a == '?') { *r = '.'; r++; }
    else if (*a == '.') { *r = '\\'; r++; *r = '.'; r++; }
    else                { *r = *a; r++; }
    a++;
}
*r = '$'; r++;
*r = 0;    // null terminator
```

## Step 2: Compile the Regular Expression

```c
char *expbuf = regcomp(reg, /* flags */);
if (expbuf == NULL) {
    perror("compile");
    return;
}
```

## Step 3: Read and Match Directory Entries

```c
DIR *dir = opendir(".");
if (dir == NULL) { perror("opendir"); return; }

struct dirent *ent;
while ((ent = readdir(dir)) != NULL) {
    if (regexec(ent->d_name, expbuf) == 0) {
        // filename matches — add to list
    }
}
closedir(dir);
```

## Step 4: Sort and Insert

**Exam question (lab3_quiz Q14):** `readdir()` returns entries in filesystem order, which is not alphabetical. Shells like `/bin/sh` sort wildcard results alphabetically. To match this behavior:

```c
// Collect matching names into a dynamically-growing array
int maxEntries = 20;
int nEntries   = 0;
char **array   = (char**)malloc(maxEntries * sizeof(char*));

while ((ent = readdir(dir)) != NULL) {
    if (regexec(ent->d_name, expbuf) == 0) {
        if (nEntries == maxEntries) {
            maxEntries *= 2;
            array = realloc(array, maxEntries * sizeof(char*));
            assert(array != NULL);
        }
        array[nEntries] = strdup(ent->d_name);
        nEntries++;
    }
}
closedir(dir);

sortArrayStrings(array, nEntries);   // alphabetical sort

for (int i = 0; i < nEntries; i++) {
    Command::_currentSimpleCommand->insertArgument(array[i]);
}
free(array);
```

## Hidden Files (Dot-Files)

In UNIX, files whose names begin with `.` (like `.bashrc`, `.login`) are "hidden" — they do not appear in `ls` output by default, and wildcards like `*` should not match them.

**Rule:** Include a filename starting with `.` in the results only if the wildcard pattern itself starts with `.`.

```c
if (regexec(ent->d_name, expbuf) == 0) {
    if (ent->d_name[0] == '.') {
        if (arg[0] == '.') {
            // Pattern starts with '.', so include dot-files
            insertArgument(strdup(ent->d_name));
        }
        // else: skip this dot-file
    } else {
        insertArgument(strdup(ent->d_name));
    }
}
```

This ensures `echo *` does not show `.` and `..`, but `echo .*` does.

## Key Terms

| Term | Meaning |
|---|---|
| Glob / wildcard | A pattern using `*` or `?` that is expanded into matching filenames |
| `opendir()` / `readdir()` / `closedir()` | POSIX API for listing directory entries |
| `regcomp()` | Compile a regex string into an internal representation |
| `regexec()` | Test a string against a compiled regex |
| Dot-file | A UNIX file whose name starts with `.`; hidden from default wildcard expansion |
