## What Makes a Command a "Builtin"?

A **builtin** command is one that the shell executes directly in the shell process itself, rather than forking a child and using `execvp`. Most commands (`ls`, `grep`, `cat`, …) are external programs on disk and require a fork + exec. Builtins are different: they must run in the parent because they need to modify the **shell's own state**.

## Why Builtins Cannot Be Forked

If a builtin ran in a child process:
1. The child would make the state change (e.g., change directory, set an environment variable).
2. The child would exit.
3. **The child's changes would be discarded** — the parent process (the shell) would be completely unaffected.

The shell's working directory and environment are per-process state managed by the kernel. A child cannot modify the parent's working directory or environment.

## The Core Builtins

### cd

```c
// NOT forked — runs in the shell process
chdir(directory);
```

`chdir()` changes the calling process's current working directory. If `cd` ran in a child, the child's directory would change but the shell's would remain unchanged.

### setenv / unsetenv

```c
// NOT forked — runs in the shell process
setenv(name, value, 1 /* overwrite */);
unsetenv(name);
```

Environment variables are inherited by child processes at `fork()` time. `setenv` modifies the calling process's environment. If it ran in a child, the change would disappear when the child exits.

### exit

```c
// NOT forked — runs in the shell process
exit(0);
```

`exit` terminates the shell itself. If it ran in a child, the child would exit but the shell would keep running.

## printenv Is the Exception

`printenv` **does** run in a forked child. The reason: its output (the list of environment variables) needs to be redirectable — the user may type `printenv > env.txt`. By running in a child, `printenv`'s stdout can be redirected without affecting the shell's stdout.

```c
// printenv — runs in child
ret = fork();
if (ret == 0) {
    // Child
    extern char **environ;
    char **p = environ;
    while (*p != NULL) {
        printf("%s\n", *p);
        p++;
    }
    exit(0);
}
// Parent waits for child
waitpid(ret, NULL, 0);
```

The `environ` global variable is a `char**` array inherited from the parent — the child has read access to all the parent's environment variables.

## Summary Table

| Command | Forked? | Reason |
|---|---|---|
| `cd` | No | `chdir()` must affect the shell process |
| `setenv` | No | `setenv()` must affect the shell's environment |
| `unsetenv` | No | `unsetenv()` must affect the shell's environment |
| `exit` | No | Must terminate the shell process |
| `printenv` | Yes | Output must be redirectable |

## Key Terms

| Term | Meaning |
|---|---|
| Builtin | A command executed in the shell process itself, not via fork+exec |
| `chdir(path)` | System call to change the calling process's current working directory |
| `setenv(name, val, overwrite)` | Add or update an environment variable in the current process |
| `environ` | Global `char**` array of all environment variables |
| Process state | Per-process data (working directory, environment, open fds) that children inherit but cannot modify in the parent |
