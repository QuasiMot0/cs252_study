## The Four Stages

Building a C program from source to executable involves four tools running in sequence:

```
.c source
   │
   ▼  gcc -E          (preprocessor)
  .i expanded source
   │
   ▼  gcc -S          (compiler + optimizer)
  .s assembly
   │
   ▼  gcc -c  (as)    (assembler)
  .o object file
   │
   ▼  gcc -o  (ld)    (linker)
 executable
```

From the slides:
> "Building: .c → preprocessor(#define/#include expands) → .i → compiler/optimizer → .s → assembler(as) → .o → linker(ld) → executable"

## Stage 1: Preprocessor

Command: `gcc -E hello.c > hello.i`

- Expands `#include` directives (inserts entire header file contents)
- Expands `#define` macros
- Processes `#ifdef`/`#endif` conditional compilation
- Output is a plain `.i` text file with all type definitions and macro expansions visible

From the slides:
> "Preprocessor: expands #include/define → full type definitions"

## Stage 2: Compiler / Optimizer

Command: `gcc -S hello.c` → produces `hello.s`

- Translates C code into x86-64 assembly
- Applies optimizations (constant folding, dead code elimination, etc.)
- Output is human-readable assembly text

## Stage 3: Assembler

Command: `gcc -c hello.c` → produces `hello.o`  
(or directly: `as hello.s -o hello.o`)

- Translates assembly mnemonics into machine code bytes
- The `.o` object file has:
  - `printf` listed as **UNDEF** (unresolved external symbol)
  - `main` at a **relative** address (not final)

From the slides:
> "Assembler output: has undefined symbols (printf=UNDEF); main has relative value"

## Stage 4: Linker

Command: `gcc -o hello hello.c`  
(or directly: `ld hello.o -o hello -lc ...`)

- Combines multiple `.o` files
- Includes needed code from static libraries (`.a`)
- Verifies that symbols from shared libraries (`.so`) exist
- Assigns final addresses
- After linking, `printf` may still be UNDEF for shared-library builds — the runtime linker resolves it at load time

From the slides:
> "After linking: printf still UNDEF until loaded by runtime linker"

## Key Terms

| Term | Meaning |
|---|---|
| Preprocessor | Expands `#include`/`#define`; output is `.i` file |
| Compiler | Translates C → assembly; output is `.s` file |
| Assembler | Translates assembly → machine code; output is `.o` object file |
| Linker (`ld`) | Combines `.o` files, resolves symbols, produces executable |
| UNDEF symbol | Symbol referenced but not yet resolved; shown in `.o` files |
| `gcc -E`, `-S`, `-c` | Stop after preprocessor, compiler, or assembler stage respectively |
