## What Is a Shared Library?

A **shared library** (`.so` on Linux, `.dylib` on macOS) is loaded into memory once and **shared across all processes** that use it. The executable does not contain the library code — only a reference to it.

From the slides:
> "Shared = one instance system-wide"
> "Shared libs: one instance in memory, mapped into multiple processes"

## The Runtime Linker (Dynamic Linker)

The loader — also called the **runtime linker** or **dynamic linker** — handles shared library loading. It runs before `main()`:

1. Allocates virtual memory sections (text/data/bss) for the executable
2. Loads the executable and all required shared libraries
3. **Resolves symbols** — writes actual addresses of shared functions into the program
4. Jumps to `_start`, which calls `init()` and then `main()`
5. When `main()` returns, `_start` calls `exit()`

From the slides:
> "Loader = runtime linker/dynamic linker; allocates sections; loads executable + shared libs; resolves symbols (writes addresses); jumps to `_start` → calls init() → calls main(); loader is at `/usr/lib/ld.so.1`"

## Advantages Over Static Libraries

| Advantage | Explanation |
|---|---|
| Smaller executables | Library code stays in the `.so`, not copied into each binary |
| Memory efficiency | One copy of library code in RAM shared by all processes |
| Easy updates | Fix a bug in the `.so` — all programs using it benefit on next launch |
| Mandatory for plugins | A shared library can be loaded at runtime without relinking |

## LD_LIBRARY_PATH

The runtime linker searches for `.so` files in standard paths (`/lib`, `/usr/lib`). To add custom paths:

```bash
export LD_LIBRARY_PATH=/my/lib/path:$LD_LIBRARY_PATH
./myprogram
```

This tells the dynamic linker where to find additional `.so` files.

## Linker Verification at Build Time

Even for shared libraries, the **static linker** (run during `gcc -o`) checks that all referenced symbols exist in the `.so`. If a symbol is not defined anywhere, you get a linker error:

From the slides:
> "If symbol undefined in executable or shared libs → linker error"

## Key Terms

| Term | Meaning |
|---|---|
| Shared library (`.so`) | Library loaded at runtime; one copy shared by all processes |
| Runtime linker / dynamic linker | Program that loads `.so` files, resolves symbols, launches `main()` |
| `/usr/lib/ld.so.1` | Path to the dynamic linker on Linux |
| `_start` | Entry point inserted by the linker; calls `main()` and then `exit()` |
| `LD_LIBRARY_PATH` | Environment variable for adding custom paths to the dynamic linker search |
