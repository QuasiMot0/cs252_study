## What Is an Executable?

From the slides:
> "A program is a file with machine instructions, initialized data, library dependencies, memory sections, and undefined values."

An executable binary contains everything the OS needs to launch a process:

- **Machine instructions** (the compiled code)
- **Initialized data** (values for global variables)
- **Section headers** describing where text/data/BSS go in memory
- **Library dependencies** (which shared libraries are required)
- **Undefined symbols** (to be resolved by the loader at runtime)

## Executable File Formats

| Format | Platform | Notes |
|---|---|---|
| **ELF** (Executable and Linkable Format) | Linux, Solaris | Current standard on UNIX-like systems |
| **COFF** (Common Object File Format) | Windows | Used in `.exe` / `.obj` files |
| **a.out** | BSD, early UNIX | Obsolete; the default output name `a.out` persists as a historical artifact |

From the slides:
> "ELF (Linux/Solaris), COFF (Windows), a.out (BSD/early UNIX, obsolete)"

## ELF Structure

ELF files contain:
- An **ELF header** with magic bytes `0x7f ELF`, architecture info, entry point address
- **Section headers**: `.text`, `.data`, `.bss`, `.rodata`, `.symtab`, `.strtab`, etc.
- **Program headers**: segments that the OS loads into memory

The `readelf` and `objdump` tools can inspect ELF files on Linux.

## Intermediate Object Files

The assembler produces `.o` (object) files — these are also ELF format. They differ from executables in that:
- Symbols like `printf` appear as `UNDEF` (unresolved)
- Code uses relative addresses, not final absolute addresses
- Multiple `.o` files are combined by the linker into a final executable

## Key Terms

| Term | Meaning |
|---|---|
| ELF | Executable and Linkable Format; standard binary format on Linux/Solaris |
| COFF | Common Object File Format; used on Windows |
| a.out | Obsolete UNIX binary format; `gcc` still defaults output name to `a.out` |
| Object file (`.o`) | Intermediate ELF with unresolved symbols; produced by the assembler |
| Undefined symbol | A function/variable referenced but not defined in this `.o`; resolved by linker |
