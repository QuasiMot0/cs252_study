## What Is a Static Library?

A **static library** (`.a` file on Linux/macOS) is an archive of `.o` object files. When you link against a static library, the linker copies the required object code **directly into your executable**.

From the slides:
> "Static libs (.a) add to exe size; shared libs (.so) only definitions"
> "Static = one instance per process"

## Linking Behavior

When the linker sees `-lm` (for example), it searches for `libm.a` and copies only the object files from the archive that define symbols actually used by your program.

Key properties:
- The resulting executable is **self-contained** — no external dependencies at runtime.
- Each process that uses the library gets its **own copy** of the code in memory.
- The executable grows in size proportional to how much of the library is used.

## Static vs Shared: Summary

| Property | Static (`.a`) | Shared (`.so`) |
|---|---|---|
| Code location | Copied into executable | Remains in separate `.so` file |
| Executable size | Larger | Smaller |
| Memory at runtime | One copy per process | One copy shared by all processes |
| Update to fix bug | Must relink all executables | Update `.so`; all programs benefit |
| Runtime dependency | None (self-contained) | Requires `.so` present at runtime |

## Static vs Dynamic Distinction

From the slides:
> "Static = events during building (static linker, static type checking C/C++); Dynamic = events at runtime (dynamic linker, dynamic type checking Python)"

"Static" in computing generally means resolved at build time; "dynamic" means resolved at run time.

## Key Terms

| Term | Meaning |
|---|---|
| Static library (`.a`) | Archive of `.o` files; code is copied into the executable at link time |
| `ar` | Unix tool to create/inspect `.a` archive files |
| Self-contained executable | Binary with all library code embedded; no runtime `.so` dependencies |
| One copy per process | Each process running a statically-linked program has its own copy of library code in RAM |
| Static linking | Resolving all symbols and embedding all needed code at build time |
