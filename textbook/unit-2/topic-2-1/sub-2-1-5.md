## What Is Dynamic Loading?

Beyond standard shared library loading (which the runtime linker handles automatically), an application can **explicitly load `.so` files at runtime** using the `dl` API. This enables plugin architectures where the program discovers and loads modules on the fly.

From the lab spec:
> "Loadable Modules: `dlopen()` to load .so, `dlsym()` to find `httprun(int ssock, char *query_string)`"

## The dl API

```c
#include <dlfcn.h>

// 1. Open the shared library
void *handle = dlopen("./mymodule.so", RTLD_LAZY);
if (!handle) {
    fprintf(stderr, "dlopen error: %s\n", dlerror());
    exit(1);
}

// 2. Get a pointer to a symbol (function or variable)
void (*httprun)(int, char*) = dlsym(handle, "httprun");
if (!httprun) {
    fprintf(stderr, "dlsym error: %s\n", dlerror());
}

// 3. Call the function
httprun(slaveSocket, queryString);

// 4. Close the library when done
dlclose(handle);
```

## `dlopen()` Flags

| Flag | Meaning |
|---|---|
| `RTLD_LAZY` | Resolve symbols only when first used |
| `RTLD_NOW` | Resolve all symbols immediately at `dlopen` time |

## `dlerror()`

After any `dl*` call, `dlerror()` returns a human-readable error string if the call failed, or `NULL` on success. Always check `dlerror()` after `dlopen` and `dlsym`.

## Lab 5 HTTP Context

In the CS252 HTTP server lab, loadable modules implement alternative request handlers. Each `.so` exports a function with this signature:

```c
void httprun(int ssock, char *query_string);
```

The server loads the module with `dlopen`, finds `httprun` with `dlsym`, and calls it. Inside `httprun`, the socket is converted to a `FILE*`:

```c
FILE *fssock = fdopen(ssock, "r+");
```

## Key Terms

| Term | Meaning |
|---|---|
| `dlopen(path, flags)` | Loads a `.so` file; returns a handle or `NULL` on failure |
| `dlsym(handle, symbol)` | Returns pointer to named symbol in the loaded library |
| `dlclose(handle)` | Unloads the shared library |
| `dlerror()` | Returns last error string from `dl*` functions |
| `RTLD_LAZY` | Lazy symbol resolution: symbols resolved on first use |
| Loadable module / plugin | A `.so` loaded at runtime via `dlopen` to extend program behavior |
