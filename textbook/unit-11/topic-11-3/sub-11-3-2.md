## HTTP Response Status Code Classes

From the slides:

| Class | Range | Meaning |
|---|---|---|
| Informational | 1XX | The request was received, continuing process |
| Successful | 2XX | The request was successfully received, understood, and accepted |
| Redirection | 3XX | Further action needs to be taken in order to complete the request |
| Client Error | 4XX | The request contains bad syntax or cannot be fulfilled |
| Server Error | 5XX | The server failed to fulfill an apparently valid request |

## Specific Status Codes Used in the Lab

### 200 — Document Follows / OK

From the slides and lab handout:

```
HTTP/1.1 200 Document follows\r\n
Server: <Server-Type>\r\n
Content-type: text/html\r\n
\r\n
<html>...</html>
```

Returned when the requested file is found and served successfully.

### 301 — Moved Permanently

From the lab handout: used when a request arrives for `/dir` (a directory path without a trailing slash). The server redirects the client to `/dir/`:

```
HTTP/1.1 301 Moved Permanently\r\n
Location: /dir/\r\n
\r\n
```

The `Location` header tells the browser where to redirect. The browser automatically re-requests the new URL.

### 401 — Unauthorized

From the slides and lab handout: returned when Basic HTTP Authentication is required but the `Authorization` header is missing or invalid:

```
HTTP/1.1 401 Unauthorized\r\n
WWW-Authenticate: Basic realm="myhttpd-cs252"\r\n
\r\n
<html>401 Unauthorized</html>
```

The `WWW-Authenticate` header tells the browser to prompt the user for a username and password. The browser then resends the request with an `Authorization` header.

### 404 — File Not Found

From the slides:

```
HTTP/1.1 404 File Not Found\r\n
Content-type: text/plain\r\n
\r\n
Could not find the URL.
```

Returned when the requested file does not exist on the server. The `Content-type` is `text/plain` even in the error case.

## Summary Table

| Code | Reason Phrase | When Used |
|---|---|---|
| 200 | Document follows | File found; body contains file content |
| 301 | Moved Permanently | Directory requested without trailing slash → redirect |
| 401 | Unauthorized | No or invalid `Authorization` header; sends `WWW-Authenticate` |
| 404 | File Not Found | Requested file does not exist |

## Key Terms

| Term | Meaning |
|---|---|
| Status code | 3-digit integer in the HTTP response status line |
| `Location` | Response header used with 3XX redirects to specify the new URL |
| `WWW-Authenticate` | Response header used with 401 to specify the authentication scheme |
| `Authorization` | Request header carrying credentials (e.g., Base64-encoded username:password) |
| Reason phrase | Human-readable text after the status code in the status line |
