## Directory Browsing

From the slides: when the requested path is a **directory** rather than a file, the server generates and returns an HTML page listing the directory's contents with clickable hyperlinks. This lets the user navigate into subdirectories and files through the browser.

From the slides (Browsing the Directory):

- **Directory**: generate hyperlinks for each subdirectory or file
- **File**: serve the file content directly
- Sorting supported by three criteria: name, size, and modification time

## Reading Directory Contents

From the lab handout: to read directory contents, use:

```c
opendir()   // open the directory
readdir()   // read each entry
stat()      // retrieve metadata: file size, modification time
```

The generated listing should match the example in `http-root-dir`, with each entry as a clickable link.

## `/dir` vs `/dir/` — The Trailing-Slash Distinction

From the lab handout: it is important to distinguish between `/dir` (no trailing slash) and `/dir/` (trailing slash):

### When the Request Is `/dir/` (with trailing slash)

From the lab handout:
> "The trailing slash signals that the client already knows this is a directory. Check whether it exists and is a directory, then serve the directory listing. If it does not exist, return a 404 Not Found response."

### When the Request Is `/dir` (without trailing slash)

From the lab handout:
> "Without the trailing slash, the path could be either a file or a directory."

- **If it is a file**: serve it normally as usual.
- **If it is a directory**: do not serve the listing directly. Instead, send an HTTP **redirect** to `/dir/`:
  ```
  HTTP/1.1 301 Moved Permanently\r\n
  Location: /dir/\r\n
  \r\n
  ```

This ensures that relative links within the directory listing resolve correctly.

## Sorting Directory Listings

From the slides and lab handout: sorting is controlled through **query parameters** in the URL:

| Query String | Sort Behavior |
|---|---|
| `?C=N;O=A` | Sort by **N**ame, **A**scending |
| `?C=N;O=D` | Sort by **N**ame, **D**escending |
| `?C=S;O=A` | Sort by **S**ize, **A**scending |
| `?C=S;O=D` | Sort by **S**ize, **D**escending |
| `?C=M;O=A` | Sort by **M**odification time, **A**scending |
| `?C=M;O=D` | Sort by **M**odification time, **D**escending |

- **C** parameter selects the column to sort by
- **O** parameter selects the order (ascending or descending)

Column headers in the browser are clickable links that append these parameters to the current URL.

## File Path Security

From the lab handout: the server must resolve file paths correctly to prevent clients from escaping the document root. Path resolution rules:

- If the request path does not include a subfolder, look inside `htdocs/`.
- Paths with subdirectories are served relative to `http-root-dir`.
- The server must **not** serve files outside the document root — paths containing `../` sequences that would escape the root must be rejected or normalized.

## Statistics and Log Pages

From the slides (Implementing the Statistics and Log pages):

### `/stats` Page

Accessible at `http://hostname:<port>/stats`. Displays:

- The names of the students who wrote the project
- The time the server has been up
- The number of requests since the server started
- The **minimum service time** and the URL request that took this time
- The **maximum service time** and the URL request that took this time

Service time = from when a connection is `accept()`ed until the socket is `close()`d. Measure with `clock()`:

```c
#include <time.h>
clock_t start, end;
double cpu_time_used;

start = clock();
/* Do the work. */
end = clock();
cpu_time_used = ((double)(end - start)) / CLOCKS_PER_SEC;
```

### `/logs` Page

Accessible at `http://hostname:<port>/logs`. Displays a list of all requests so far. Each line includes:

- The **source host** of the request
- The **directory** requested

The log is written to a **file on disk** so it is preserved across server restarts. Every time the server starts, it **appends** to the existing log file rather than overwriting it.

## Key Terms

| Term | Meaning |
|---|---|
| Directory listing | HTML page generated by server listing directory contents with links |
| `opendir()`/`readdir()` | System calls for reading directory entries |
| `stat()` | System call to retrieve file metadata (size, modification time) |
| Trailing slash | `/dir/` → serve directory listing; `/dir` → redirect if it is a directory |
| 301 redirect | Moves client from `/dir` to `/dir/`; `Location` header gives new URL |
| Directory traversal | Attack where `../` sequences in the path escape the document root |
| `/stats` | Special URL returning server uptime, request count, and min/max service times |
| `/logs` | Special URL returning per-request log (source host, path); persisted to disk |
