## What Is CGI-BIN?

From the slides:

> "CGI-BIN: Proxy the request to a backend script."

CGI (Common Gateway Interface) allows the web server to execute scripts and programs and send their output back to the client. When a request arrives for a path under `/cgi-bin/`, the server does **not** serve a static file — instead it forks a child process, sets up the environment, and executes the script.

## CGI GET Request Format

From the slides:

```
GET /cgi-bin/<script>?{<var>=<val>&}*{<var>=<val>} HTTP/1.1\r\n
{<Other Header Information>\r\n}*
\r\n
```

- `<script>`: an executable program in the `cgi-bin/` directory
- **Query string**: `?{<var>=<val>&}*{<var>=<val>}` — everything after the `?` in the URL
- Example: `GET /cgi-bin/test-cgi?name=Alejandro&age=22&country=Spain HTTP/1.1`

## CGI-BIN Implementation Steps

From the slides (Implementation of CGI-BIN):

1. **Fork** a child process.
2. Set the environment variable `REQUEST_METHOD=GET`.
3. Set the environment variable `QUERY_STRING=<everything after the ? in the URL>`.
4. **Redirect** the output of the child process to the slave socket (so the script's stdout goes directly to the client).
5. Send the following header **before** the script output:
   ```
   HTTP/1.1 200 Document follows\r\n
   Server: <Server-Type>\r\n
   ```
6. Call **`execvp()`** to execute the script.

**Note**: the script itself prints the `Content-Type` header and generates the response body. These are sent directly to the browser via the redirected stdout.

## Environment Variables

From the slides and lab handout:

| Variable | Value |
|---|---|
| `REQUEST_METHOD` | `"GET"` (or `"POST"` for extra credit) |
| `QUERY_STRING` | Everything after the `?` in the URL (e.g., `name=Alejandro&age=22`) |

The script is responsible for decoding the query string.

## Passing Variables: GET vs POST

From the lab handout:

- **GET method**: query string (everything after the `?`) is passed to the script as `QUERY_STRING`. Must also set `REQUEST_METHOD` to `"GET"`.
- **POST method**: extra credit only. Variables are passed in the request body instead of the URL.

## Loadable Modules (`.so` files)

From the slides (Loadable Modules):

Instead of a traditional CGI script, the server can load a **shared library** (`.so` file) directly:

```
GET /cgi-bin/hello.so?a=b HTTP/1.1
```

Implementation:
1. Call `dlopen()` to load the dynamic library `hello.so`.
2. Call `dlsym()` to look up the symbol `httprun`:
   ```c
   extern "C" httprun(int ssock, char * query_string);
   ```
3. Call `httprun()` directly.

Hints from the slides:
```c
FILE * fssock = fdopen(ssock, "r+");
fprintf(fssock, "tomato, and mayo.<P>%c", LF);
fclose(fssock);
```

Use `ssock` as a file descriptor and `fdopen()` to wrap it in a `FILE *` for `fprintf`.

## Key Terms

| Term | Meaning |
|---|---|
| CGI | Common Gateway Interface; standard for running scripts from a web server |
| `QUERY_STRING` | Environment variable containing URL parameters after the `?` |
| `REQUEST_METHOD` | Environment variable set to `"GET"` (or `"POST"`) |
| `execvp()` | Replace child process image with the script executable |
| Loadable module | `.so` shared library loaded at runtime via `dlopen()`/`dlsym()` |
| `dlopen()` | Opens a shared library for dynamic loading |
| `dlsym()` | Looks up a symbol (function) in a loaded shared library |
| `httprun()` | Convention for the entry point function in a CGI loadable module |
