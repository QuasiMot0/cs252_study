## What Is the Internet?

From the slides: the Internet is a **collection of networks, routers, and hosts**. A **virtual network** hides the heterogeneity of the underlying hardware, so any two hosts can communicate regardless of what physical technology connects them.

## TCP/IP Layering

From the slides:

| Layer | Examples |
|---|---|
| Application | HTTP, FTP, SSH |
| Transport | TCP, UDP |
| Internet | IP |
| Network Interface | Ethernet, Wi-Fi |
| Physical | Cables, radio |

Each layer adds its own header; lower layers carry upper-layer packets as payloads.

## IP Addresses

From the slides:

- IP addresses are **32 bits** long, written in dotted-decimal notation (e.g., `128.10.3.24`).
- An IP address is an **abstraction** — it hides the hardware address (MAC address) of the network interface.
- An IP address identifies a **network interface**, not a host; a host with two NICs has two IP addresses.

### Prefix and Suffix

From the slides: an IP address has two parts:

- **Prefix** = network number (identifies which network the host is on)
- **Suffix** = host number (identifies the host within that network)

The **subnet mask** indicates how many leading bits are the prefix. Example: mask `255.255.255.0` means the first 24 bits are the network prefix.

## ARP — Address Resolution Protocol

From the slides: ARP translates an **IP address** into a **hardware (MAC) address** so that frames can be delivered on a local network.

- A host broadcasts "who has IP X?" and the owner of IP X replies with its MAC address.
- ARP results are cached in the ARP table to avoid repeated broadcasts.
- The `arp` command shows the local ARP table.

## Routing

From the slides:

- Every router and host has a **routing table** that maps destination prefixes to **next-hop** addresses.
- Routing tables can be configured **manually** or populated automatically by routing protocols.

### What Changes Across Hops

From the slides (IP packet from A to M example):

| Field | Behavior |
|---|---|
| IP source address | **Unchanged** end-to-end |
| IP destination address | **Unchanged** end-to-end |
| Hardware (MAC) source | **Changes** at every hop |
| Hardware (MAC) destination | **Changes** at every hop |

The IP layer provides the stable end-to-end addressing; the network-interface layer handles local delivery one hop at a time.

## DNS — Domain Name System

From the slides: DNS **translates hostnames into IP addresses**. Lookups are distributed across a hierarchy of name servers.

- Hostname format: `host.subdomain.domain.tld` — e.g., `ector.cs.purdue.edu`
- A DNS lookup for `ector.cs.purdue.edu` contacts the root → `.edu` → `purdue.edu` → `cs.purdue.edu` servers in turn.

## DHCP — Dynamic Host Configuration Protocol

From the slides: DHCP **automatically assigns** configuration to a host when it joins a network. A DHCP server provides:

- Local IP address
- Subnet mask
- Default router (gateway)
- DNS server address

Commands to view the assigned configuration:
- UNIX: `ifconfig`
- Windows: `ipconfig`

## Key Terms

| Term | Meaning |
|---|---|
| IP address | 32-bit identifier for a network interface; written in dotted-decimal |
| Prefix | Network portion of an IP address |
| Suffix | Host portion of an IP address |
| Subnet mask | Indicates how many bits are the network prefix |
| ARP | Protocol that maps IP address to hardware (MAC) address |
| Routing table | Maps destination prefixes to next-hop addresses |
| DNS | Distributed system translating hostnames to IP addresses |
| DHCP | Protocol that auto-assigns IP, mask, gateway, and DNS to new hosts |
