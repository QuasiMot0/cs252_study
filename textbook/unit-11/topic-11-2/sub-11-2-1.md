## Sockets and Ports

From the slides:

- A **socket** is a **file descriptor** that represents a network connection. Reading and writing to a socket sends and receives data over the network.
- A **port** identifies an application endpoint on a host — multiple applications on the same machine use different ports.
- **Well-known ports**: HTTP = 80, HTTPS = 443, SSH = 22, FTP = 21
- Ports below 1024 require **root privilege** to bind
- Valid port range for user programs: **1024 – 65535** (2¹⁶ − 1)
- Sockets originated from **UNIX BSD**
- Two socket types: **stream** (TCP) and **message/datagram** (UDP)

## Server Socket Lifecycle

From the lab slides (Daytime Server code):

### Step 1: Set Up Address Structure

```c
int port = atoi(argv[1]);

struct sockaddr_in serverIPAddress;
memset(&serverIPAddress, 0, sizeof(serverIPAddress));
serverIPAddress.sin_family = AF_INET;
serverIPAddress.sin_addr.s_addr = INADDR_ANY;   // accept connections on any interface
serverIPAddress.sin_port = htons((u_short) port);
```

- `INADDR_ANY`: bind to all local interfaces
- `htons()`: converts port number to **network byte order** (big-endian)

### Step 2: Create the Socket

```c
int masterSocket = socket(PF_INET, SOCK_STREAM, 0);
if (masterSocket < 0) {
    perror("socket");
    exit(-1);
}
```

- `PF_INET`: IPv4 protocol family
- `SOCK_STREAM`: TCP (stream) socket

### Step 3: Set SO_REUSEADDR

```c
int optval = 1;
int err = setsockopt(masterSocket, SOL_SOCKET, SO_REUSEADDR,
                     (char *) &optval, sizeof(int));
```

Without `SO_REUSEADDR`, after a server exits you must wait ~2 minutes before reusing the same port (due to TIME_WAIT state). This option allows immediate reuse.

### Step 4: Bind

```c
int error = bind(masterSocket,
                 (struct sockaddr *) &serverIPAddress,
                 sizeof(serverIPAddress));
if (error) {
    perror("bind");
    exit(-1);
}
```

Assigns the IP address and port to the socket.

### Step 5: Listen

```c
int QueueLength = 5;
error = listen(masterSocket, QueueLength);
if (error) {
    perror("listen");
    exit(-1);
}
```

Puts the socket in passive/listening mode. `QueueLength` is the maximum number of pending connections queued before `accept()` is called.

### Step 6: Accept Loop

```c
while (1) {
    struct sockaddr_in clientIPAddress;
    int alen = sizeof(clientIPAddress);
    int slaveSocket = accept(masterSocket,
                             (struct sockaddr *) &clientIPAddress,
                             (socklen_t *) &alen);
    if (slaveSocket < 0) {
        perror("accept");
        exit(-1);
    }

    // Process the request using slaveSocket
    processRequest(slaveSocket);

    // Close the connection socket (not the master socket)
    close(slaveSocket);
}
```

- `accept()` **blocks** until a client connects; returns a new socket descriptor (`slaveSocket`) for communicating with that specific client
- The `masterSocket` continues listening for new clients
- `clientIPAddress` is filled with the client's IP and port
- After processing, `close(slaveSocket)` ends the connection

## Summary of the Lifecycle

From the slides:

```
socket()        → create master socket (file descriptor)
setsockopt()    → allow port reuse (SO_REUSEADDR)
bind()          → associate socket with IP address and port
listen()        → put socket in passive mode, set queue depth
accept()        → block until client connects; return slave socket
read()/write()  → exchange data with client via slave socket
close()         → close slave socket (one per connection)
```

## Reading from the Client (Character-by-Character)

From the lab slides (Daytime Server `processTimeRequest`): HTTP headers are terminated by `\r\n` (carriage return + linefeed). Read one byte at a time until `\r\n` is detected:

```c
// Read until <CR><LF>
while (nameLength < MaxName &&
       (n = read(fd, &newChar, sizeof(newChar))) > 0) {
    if (lastChar == '\015' && newChar == '\012') {
        nameLength--;   // discard the <CR> from the buffer
        break;
    }
    name[nameLength] = newChar;
    nameLength++;
    lastChar = newChar;
}
name[nameLength] = 0;   // null-terminate
```

## Key Terms

| Term | Meaning |
|---|---|
| Socket | File descriptor for a network connection |
| Port | Numeric identifier for an application endpoint on a host |
| masterSocket | Listening socket; accepts new clients; never used for data |
| slaveSocket | Per-connection socket returned by `accept()`; used for data |
| `INADDR_ANY` | Bind to all available network interfaces |
| `SO_REUSEADDR` | Allow port reuse immediately after server exits |
| `htons()` | Convert host byte order to network byte order (big-endian) |
| `QueueLength` | Maximum pending connections before `accept()` is called |
