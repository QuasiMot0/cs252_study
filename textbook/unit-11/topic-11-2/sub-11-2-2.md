## Client Socket Lifecycle

From the slides (Client for Daytime Server), the client side of a TCP connection follows these steps:

### Step 1: Create the Socket

```c
int sock = socket(PF_INET, SOCK_STREAM, proto);
```

- `PF_INET`: IPv4 protocol family
- `SOCK_STREAM`: TCP connection

### Step 2: Resolve the Hostname and Set Up Address

```c
struct sockaddr_in socketAddress;
memset(&socketAddress, 0, sizeof(socketAddress));
socketAddress.sin_family = AF_INET;

// Look up hostname → IP address
struct hostent *host = gethostbyname(argv[1]);
socketAddress.sin_addr = *((struct in_addr *) host->h_addr);

// Convert port to network byte order
socketAddress.sin_port = htons((u_short) atoi(argv[2]));
```

- `gethostbyname()`: DNS lookup — converts a hostname (e.g., `ector.cs.purdue.edu`) to an IP address
- `htons()`: converts the port number from host byte order to network byte order

### Step 3: Connect to the Server

```c
int error = connect(sock,
                    (struct sockaddr *) &socketAddress,
                    sizeof(socketAddress));
if (error) {
    perror("connect");
    exit(-1);
}
```

`connect()` performs the TCP **3-way handshake** (SYN → SYN+ACK → ACK). After it returns, the connection is established.

### Step 4: Send a Request

```c
// Write to the socket just like a file
write(sock, request, strlen(request));
```

### Step 5: Receive the Reply

```c
char buf[1000];
int n = recv(sock, buf, sizeof(buf), 0);
while (n > 0) {
    // Write n bytes to stdout
    write(1, buf, n);
    // Continue receiving
    n = recv(sock, buf, sizeof(buf), 0);
}
```

The client reads data in a loop because TCP is a stream — multiple `recv()` calls may be needed to receive the full response.

### Step 6: Close the Socket

```c
closesocket(sock);
exit(0);
```

Closing the socket sends a FIN to the server, signaling end of communication.

## Summary of the Lifecycle

From the slides:

```
socket()          → create socket (file descriptor)
gethostbyname()   → resolve hostname to IP address
htons()           → convert port to network byte order
connect()         → establish TCP connection (3-way handshake)
write()/read()    → send request; receive response
close()           → close the connection
```

## Comparison: Client vs Server

| Step | Server | Client |
|---|---|---|
| Create socket | `socket()` | `socket()` |
| Bind address | `bind()` | (not needed) |
| Set passive mode | `listen()` | — |
| Establish connection | `accept()` (waits) | `connect()` (initiates) |
| Exchange data | `read()`/`write()` on slaveSocket | `recv()`/`write()` on sock |
| Close | `close(slaveSocket)` | `closesocket(sock)` |

## Key Terms

| Term | Meaning |
|---|---|
| `gethostbyname()` | Resolves a hostname to an IP address via DNS |
| `connect()` | Initiates a TCP connection; performs 3-way handshake |
| `recv()` | Receive data from a socket; may return less than the full response |
| `htons()` | Host-to-network short: converts port to big-endian byte order |
| `sockaddr_in` | Structure holding IPv4 address family, IP, and port |
| `closesocket()` | Closes the socket and sends FIN to signal end of connection |
