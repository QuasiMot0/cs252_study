## egrep for Pattern Matching in Scripts

From the lab:
> "To prevent output from egrep either redirect the output to a trash file that your program will clean up after running using the rm command or use egrep with the -q option."

`egrep` (extended grep) is the standard tool for regex testing in bash scripts. The `-q` flag suppresses all output and just sets the exit code:

```bash
if echo "$password" | egrep -q '[A-Z]'
then
  echo "Has uppercase"
fi
```

Exit code 0 = pattern found; exit code 1 = not found. This allows egrep to drive `if` conditions directly.

## Common Regex Character Classes

| Pattern | Matches |
|---|---|
| `[A-Z]` | Any uppercase letter |
| `[a-z]` | Any lowercase letter |
| `[A-Za-z]` | Any letter |
| `[0-9]` | Any digit |
| `[#$+%@^*-/]` | Any of these special characters |
| `.` | Any single character |

## Quantifiers

| Quantifier | Meaning |
|---|---|
| `*` | Zero or more of the preceding |
| `+` | One or more of the preceding |
| `?` | Zero or one of the preceding |
| `{3}` | Exactly 3 of the preceding |
| `{3,}` | 3 or more of the preceding |

## Lab 2 Regex Patterns (Password Checker)

From the lab, password validation uses patterns like:

| Check | Regex | Action |
|---|---|---|
| Has uppercase | `[A-Z]` | required |
| Has lowercase | `[a-z]` | required |
| Has digit | `[0-9]` | required; +2 per digit |
| Has special char | `[#$+%@^*-/]` | required; +2 per match |
| Repeated alphanumeric | `([a-zA-Z0-9])\1` | -10 points |
| 3+ consecutive lowercase | `[a-z]{3}` | -3 points |
| 3+ consecutive uppercase | `[A-Z]{3}` | -3 points |
| 3+ consecutive digits | `[0-9]{3}` | -3 points |

Example pattern check:
```bash
if echo "$password" | egrep -q '[a-z]{3}'
then
  score=`expr $score - 3`
fi
```

## awk for Field Extraction and Counting

`awk` splits input into fields by whitespace (by default). Fields are accessed as `$1`, `$2`, etc.

```bash
# Extract second field (month) from date output
MONTH=$(date | awk '{print $2}')

# Count characters matching a pattern
count=$(echo "$password" | awk '{n=gsub(/[0-9]/,""); print n}')
```

`awk '{print length($0)}'` prints the length of the entire line — used to measure password length.

## sed for Substitution

`sed` (stream editor) applies substitutions:

```bash
# Remove all digits from a string
echo "$password" | sed 's/[0-9]//g'

# Count occurrences by deleting non-matching chars and measuring length
digits=$(echo "$password" | sed 's/[^0-9]//g')
```

## Key Terms

| Term | Meaning |
|---|---|
| `egrep -q pattern` | Test if pattern matches; returns exit code (no output) |
| `[A-Z]` | Character class: any uppercase letter |
| `{3}` | Quantifier: exactly 3 repetitions |
| `\1` | Backreference to first capture group (used for repeated chars) |
| `awk '{print $2}'` | Print second whitespace-delimited field |
| `awk '{print length($0)}'` | Print character count of line |
| `sed 's/pattern/replace/g'` | Global substitution |
