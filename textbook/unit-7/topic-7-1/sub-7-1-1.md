## The Shebang Line

Every bash script starts with:

```bash
#!/bin/bash
```

This tells the OS which interpreter to use. The script must be made executable (`chmod u+x script.sh`) before it can be run as `./script.sh`.

## Variables

Variables are assigned with `=` â€” **no spaces** around the `=` sign:

```bash
count=0
dirs=$*
name="Alice"
```

Variables are accessed with `$`:

```bash
echo $count
echo "$name"
```

From the slides (hello.sh example):
```bash
#!/bin/bash
echo "Hello $USER"
echo "Welcome to CS252"
echo "Your home directory is \"$HOME\""
echo "And your current directory is \"$PWD\""
```

Common built-in environment variables available in scripts:

| Variable | Value |
|---|---|
| `$USER` | Current login name |
| `$HOME` | Home directory |
| `$PWD` | Current working directory |
| `$PATH` | Executable search path |

## Command Substitution

To capture the output of a command into a variable, use `$(command)` or backticks `` `command` ``:

```bash
# Modern form (preferred, nestable)
MONTH=$(date | awk '{print $2}')

# Old form (backtick)
count=`expr $count + 1`
```

From the slides (mail-hello.sh):
```bash
echo "Today is" `date` >> tmp-message
```

From the Lab 2 quiz, `MONTH=$(date | awk '{print $2}')` correctly stores the second field (month) from `date` output.

## Arithmetic

Bash variables are strings by default. Use `expr` for arithmetic:

```bash
count=`expr $count + 1`
```

Or use `$(( ))`:

```bash
count=$((count + 1))
```

## Quoting Rules

| Quoting | Effect |
|---|---|
| `"double quotes"` | Variables and command substitution **are** expanded |
| `'single quotes'` | Everything is **literal** â€” no expansion at all |
| `\` | Escape next character (e.g., `\"` inside double quotes) |

```bash
echo "Hello $USER"      # prints: Hello grr
echo 'Hello $USER'      # prints: Hello $USER (literal)
echo "It is \"$PWD\""   # prints: It is "/homes/grr"
```

## Key Terms

| Term | Meaning |
|---|---|
| `#!/bin/bash` | Shebang; tells OS to run this script with bash |
| `VAR=value` | Assign a variable (no spaces around `=`) |
| `$VAR` | Expand a variable's value |
| `$(command)` | Command substitution: captures stdout of command |
| `` `command` `` | Older backtick form of command substitution |
| `expr` | Evaluate arithmetic expression and print result |
| Double quotes | Allow variable and command expansion |
| Single quotes | Suppress all expansion (literal string) |
