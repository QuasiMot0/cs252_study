## Positional Parameters

When a script is invoked, the shell sets special variables for each argument:

| Variable | Meaning |
|---|---|
| `$0` | The name of the script itself |
| `$1` | First argument |
| `$2` | Second argument |
| `$N` | Nth argument |
| `$#` | Number of arguments (does **not** count `$0`) |
| `$*` | All arguments as a single string |
| `$@` | All arguments as separate quoted strings |

From the slides (hello-loop.sh):
```bash
#!/bin/bash
if [ $# -lt 1 ]
then
  echo "$0 needs at least one argument"
  echo "  Eg."
  echo "      $0 Mickey Donald Daisy"
  exit 1
fi

for friend in $*
do
  echo "Hello $friend"
done
```

Running `./hello-loop.sh Mickey Donald Daisy`:
- `$0` = `./hello-loop.sh`
- `$1` = `Mickey`, `$2` = `Donald`, `$3` = `Daisy`
- `$#` = 3
- `$*` = `Mickey Donald Daisy`

## Parsing Flags with getopts / manual shift

From the lab, flags like `-f passwordFile` or `-t secs` are parsed manually:

```bash
# Usage: ./pwcheck.sh [-f passwordFile] [password1 password2 ...]
if [ "$1" = "-f" ]
then
  passwordFile=$2
  shift 2          # remove -f and filename from positional params
fi
```

`shift N` removes the first N positional parameters and renumbers the rest.

## test and [ ] Expressions

`[ condition ]` is equivalent to calling `test condition`. Spaces inside the brackets are required.

### Numeric comparisons

| Operator | Meaning |
|---|---|
| `-lt` | Less than |
| `-gt` | Greater than |
| `-le` | Less than or equal |
| `-ge` | Greater than or equal |
| `-eq` | Equal |
| `-ne` | Not equal |

```bash
if [ $# -lt 1 ]     # fewer than 1 argument
if [ $count -gt 10 ]
```

### String comparisons

| Operator | Meaning |
|---|---|
| `=` | String equal |
| `!=` | String not equal |
| `-z "$s"` | String is empty |
| `-n "$s"` | String is non-empty |

```bash
if [ "$1" = "-f" ]
if [ -z "$password" ]
```

### File tests

| Operator | Meaning |
|---|---|
| `-f file` | Exists and is a regular file |
| `-d dir` | Exists and is a directory |
| `-e path` | Exists (any type) |

```bash
if [ ! -d "$backupdir" ]
then
  mkdir "$backupdir"
fi
```

## Exit Codes

```bash
exit 0     # success
exit 1     # generic error
```

The last command's exit code is in `$?`:

```bash
egrep -q 'pattern' file
if [ $? -eq 0 ]
then
  echo "found"
fi
```

Or more directly:
```bash
if egrep -q 'pattern' file
then
  echo "found"
fi
```

## Key Terms

| Term | Meaning |
|---|---|
| `$0` | Script name |
| `$1` ... `$N` | Positional arguments |
| `$#` | Argument count (not including `$0`) |
| `$*` | All arguments as one string |
| `$@` | All arguments as separate strings |
| `[ condition ]` | Test expression (alias for `test`) — spaces required |
| `-lt`, `-gt`, `-eq` | Numeric comparison operators |
| `=`, `!=`, `-z`, `-n` | String comparison/test operators |
| `-f`, `-d`, `-e` | File existence tests |
| `shift N` | Remove first N positional parameters |
| `exit N` | Exit with code N (0=success, non-zero=error) |
| `$?` | Exit code of last command |
