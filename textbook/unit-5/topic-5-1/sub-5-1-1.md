## What Is gdb?

From the slides:
> "GDB is a debugger that helps you debug your program."
> "The time you spend now learning gdb will save you days of debugging time."
> "A debugger will make a good programmer a better programmer."

## Compiling for gdb

You must compile with `-g` to embed debugging information (source file names, line numbers, variable names) into the executable:

From the slides:
> "You need to compile with the '-g' option to be able to debug a program with gdb. The '-g' option adds debugging information to your program."

```bash
gcc -g -o hello hello.c
```

## Launching gdb

```bash
gdb progname
(gdb)
```

## Setting Breakpoints

A breakpoint is a marker that makes the program stop and return control to gdb.

From the slides, three forms:

| Command | Effect |
|---|---|
| `(gdb) break main` | Break at the start of `main` |
| `(gdb) break 66` | Break at line 66 of the current file |
| `(gdb) break hello.c:78` | Break at line 78 of `hello.c` |

## Running the Program

```
(gdb) run
```
- Starts the program; runs until a breakpoint is hit.
- Pass arguments after `run`: `(gdb) run arg1 arg2`
- If the program runs without stopping, press **ctrl-c** to regain control.

From the slides:
> "Your program will start running and when it reaches 'main()' it will stop."

## Stepping Through Code

| Command | Effect |
|---|---|
| `(gdb) step` | Run the next line; **enters** function calls |
| `(gdb) next` | Run the next line; **steps over** function calls |

From the slides:
> "`step` — It will run the next line of code and stop. If it is a function call, it will enter into it."
> "`next` — If it is a function call, it will not enter the function and it will go through it."

## Printing Variables

```
(gdb) print var
```

Examples from the slides:
```
(gdb) print i
$1 = 5

(gdb) print s1
$1 = 0x10740 "Hello"

(gdb) print stack[2]
$1 = 56

(gdb) print stack
$2 = {0, 0, 56, 0, 0, 0, 0, 0, 0, 0}
```

## Backtrace (where)

```
(gdb) where
```

Prints the current function and the chain of callers (the call stack):

```
(gdb) where
#0  main () at test_mystring.c:22
(gdb)
```

## Exiting gdb

```
(gdb) quit
The program is running. Exit anyway? (y or n) y
```

## Key Terms

| Term | Meaning |
|---|---|
| `-g` flag | Embeds debug symbols; required for gdb |
| Breakpoint | Marker that pauses execution and returns control to gdb |
| `break function` | Set breakpoint at function entry |
| `break file:line` | Set breakpoint at specific file and line |
| `step` | Execute next line; enters function calls |
| `next` | Execute next line; skips over function calls |
| `print var` | Display current value of a variable or expression |
| `where` / `bt` | Print the current call stack (backtrace) |
| ctrl-c | Interrupt a running program inside gdb |
