## What Is a Thread?

From the slides:
> "A thread is a path of execution."
> "By default, a C program has only one thread called the 'main thread' which starts running main()."

A process can have multiple threads. Each thread is an independent path of execution through the program's code, but all threads within a process share the same address space.

## Private State (Per-Thread)

From the slides:
> "Each thread has its own: Stack, PC (Program Counter), Registers, State, Local variables and function calls"

| Private to each thread | Why |
|---|---|
| Stack | Each thread has its own function call chain |
| Program Counter (PC) | Each thread is at a different point in execution |
| Registers | CPU register values belong to the currently executing thread |
| State (Ready/Running/Waiting) | Each thread is scheduled independently |
| Local variables | Stored on the thread's own stack |

## Shared State (Across All Threads)

From the slides:
> "All threads in the same process share: Heap, Global variables, Code"

| Shared across all threads | Why |
|---|---|
| Heap | `malloc()`/`free()` memory is accessible to all threads |
| Global variables | Declared outside any function; all threads can read and write them |
| Code (text segment) | All threads run the same program; they share the same instructions |

## How the OS Tracks Threads

From the slides (process table entry):
> "Per thread: PC (Program Counter), Registers, Stack, State (Wait/Ready/Running)"

The process table stores per-thread data (PC, registers, stack, state). Everything else in the process table entry (PID, environment, open files, memory mappings) is shared across all threads in that process.

## Key Terms

| Term | Meaning |
|---|---|
| Thread | A path of execution within a process |
| Main thread | Default thread that runs `main()` when a C program starts |
| Private thread state | Stack, PC, registers, state — unique to each thread |
| Shared thread state | Heap, global variables, code — common to all threads in the process |
