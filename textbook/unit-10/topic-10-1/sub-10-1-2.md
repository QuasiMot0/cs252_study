## Creating a Thread: pthread_create

From the slides:
> "pthread_create(&thr_id, attr, func, arg)"

```c
pthread_t thr_id;
pthread_create(&thr_id, NULL, myfunction, myarg);
```

| Parameter | Meaning |
|---|---|
| `&thr_id` | Pointer to a `pthread_t` variable; receives the new thread's ID |
| `attr` | Thread attributes (pass `NULL` for defaults) |
| `func` | Function the thread will execute |
| `arg` | Argument passed to `func` (cast to `void*`) |

The thread function must have the signature:
```c
void* func(void* arg);
```

## Waiting for a Thread: pthread_join

From the slides:
> "pthread_join(&t1) // wait for t1 to finish"

```c
pthread_join(thr_id, NULL);
```

`pthread_join` blocks the calling thread until the specified thread finishes execution. It is the thread equivalent of `waitpid()` for processes.

## Multi-Threaded Program Example

From the slides:

```c
#include <pthread.h>

void* prstr(void* str) {
    int i;
    for (i = 0; i < 100; i++) {
        printf("%s\n", (char*)str);
    }
}

int main() {
    pthread_t t1, t2;
    pthread_create(&t1, NULL, prstr, "b");
    pthread_create(&t2, NULL, prstr, "c");
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
}
```

From the slides, the output interleaves "b" and "c" in an unpredictable order:
```
b
c
b
b
c
...
```

The interleaving is non-deterministic because the scheduler decides when to context-switch between threads.

## Key Terms

| Term | Meaning |
|---|---|
| `pthread_t` | Type for a thread ID |
| `pthread_create` | Creates a new thread; takes thread ID pointer, attrs, function, and argument |
| `pthread_join` | Blocks caller until the specified thread finishes |
| Thread function | Must have signature `void* func(void* arg)` |
