## Applications of Threads

From the slides:
> "Concurrent server: a web server may need to handle a fast client and a slow client at the same time. If the server handles only one at a time, the slow client will slow down the fast client."
> "Multiple CPUs: threads on different CPUs can speed up the program."
> "Interactive application: one thread handles user input; another updates the display."

## Advantages of Threads Over Processes

From the slides:
> "Creating a new thread is much faster than creating a new process: Creating a new address space and a new open file table is expensive."
> "Context switch between threads is faster because we do not have to switch to a new address space."
> "Communication between threads using global variables is fast."

| Advantage | Reason |
|---|---|
| Fast creation | No new address space or open file table; only a new stack and PC |
| Fast context switch | No address space switch — threads share the same memory |
| Fast communication | Shared heap and global variables; no IPC (pipes, sockets) needed |

## Disadvantages of Threads vs Processes

From the slides:
> "Threads are less robust: if one thread crashes (for example null pointer exception), all threads in the process will die."
> "Multiple threads need to coordinate accesses to global variables using mutex, semaphores..."

| Disadvantage | Explanation |
|---|---|
| Less robust | A crash in any one thread (e.g., segfault) kills the entire process and all its threads |
| Synchronization required | Shared globals create race conditions; threads must use mutexes or semaphores |

## Summary Comparison

| Property | Processes | Threads |
|---|---|---|
| Address space | Separate | Shared |
| Creation cost | High | Low |
| Context switch cost | High | Low |
| Communication | IPC (expensive) | Shared globals (fast) |
| Robustness | Isolated (one crash doesn't affect others) | One crash kills all threads |
| Synchronization | Not needed for private state | Required for shared globals |

## Key Terms

| Term | Meaning |
|---|---|
| Concurrent server | Server using threads to handle multiple clients simultaneously |
| Race condition | Unpredictable result when threads access shared state without coordination |
| Mutex / Semaphore | Synchronization primitives used to protect shared state between threads |
