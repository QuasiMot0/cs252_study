## What Is a Mutex?

From the slides:
> "A mutex (mutual exclusion lock) is a lock that ensures only one thread can execute the critical section at a time."

A mutex has two states: **locked** and **unlocked**. When a thread tries to lock a mutex that is already locked, it **blocks** (waits) until the mutex is unlocked.

## POSIX Mutex API

```c
pthread_mutex_t mutex;

pthread_mutex_init(&mutex, NULL);   // initialize (NULL = default attributes)

pthread_mutex_lock(&mutex);         // acquire the lock (blocks if already locked)
// --- critical section ---
pthread_mutex_unlock(&mutex);       // release the lock
```

| Function | Effect |
|---|---|
| `pthread_mutex_init(&mutex, attr)` | Initializes the mutex; pass `NULL` for default attributes |
| `pthread_mutex_lock(&mutex)` | Acquires the lock; blocks if another thread holds it |
| `pthread_mutex_unlock(&mutex)` | Releases the lock; unblocks one waiting thread |

## Fixing the Counter Race Condition

From the slides:

```c
pthread_mutex_t mutex;

void* count(void* arg) {
    int i;
    for (i = 0; i < 100; i++) {
        pthread_mutex_lock(&mutex);
        counter++;                    // critical section
        pthread_mutex_unlock(&mutex);
    }
}

int main() {
    pthread_mutex_init(&mutex, NULL);
    // ... create threads, join threads ...
}
```

With the mutex, only one thread executes `counter++` at a time, eliminating the lost update.

## How It Works

1. Thread 1 calls `pthread_mutex_lock` — lock is free, so Thread 1 acquires it.
2. Thread 2 calls `pthread_mutex_lock` — lock is held by Thread 1, so Thread 2 **blocks**.
3. Thread 1 calls `pthread_mutex_unlock` — lock is released.
4. Thread 2 unblocks and acquires the lock.

## Key Terms

| Term | Meaning |
|---|---|
| Mutex | Mutual exclusion lock; ensures only one thread enters the critical section at a time |
| `pthread_mutex_t` | POSIX type for a mutex variable |
| `pthread_mutex_lock` | Acquires the mutex; blocks if it is already held |
| `pthread_mutex_unlock` | Releases the mutex; allows a blocked thread to proceed |
| Blocking | Thread is put to sleep until the mutex becomes available |
