## What Is a Bounded Buffer?

From the slides:
> "A bounded buffer is a circular queue of fixed maximum size."

Producers add items to the buffer; consumers remove items. The buffer:
- Blocks a **producer** if it is full (no space to insert)
- Blocks a **consumer** if it is empty (no item to remove)

## Class Definition

From the slides:

```c
class BoundedBuffer {
    int _buf[MaxSize];       // circular array
    int _head, _tail;        // front and back indices
    int _size;               // current number of items

    Semaphore _emptySem;     // count = # items in buffer (init 0)
    Semaphore _fullSem;      // count = # empty spaces  (init MaxSize)
    Mutex _mutex;
};

BoundedBuffer() {
    _head = _tail = _size = 0;
    sema_init(&_emptySem, 0);
    sema_init(&_fullSem, MaxSize);
}
```

| Semaphore | Initial Value | Meaning |
|---|---|---|
| `_emptySem` | 0 | Number of items currently in the buffer |
| `_fullSem` | MaxSize | Number of empty slots currently in the buffer |

## enqueue (Producer)

From the slides:

```c
void enqueue(int val) {
    sema_wait(&_fullSem);       // wait for an empty space
    mutex_lock(&_mutex);
    _buf[_tail] = val;
    _tail = (_tail + 1) % MaxSize;
    _size++;
    mutex_unlock(&_mutex);
    sema_post(&_emptySem);      // signal: one more item available
}
```

## dequeue (Consumer)

From the slides:

```c
int dequeue() {
    sema_wait(&_emptySem);      // wait for an item
    mutex_lock(&_mutex);
    int val = _buf[_head];
    _head = (_head + 1) % MaxSize;
    _size--;
    mutex_unlock(&_mutex);
    sema_post(&_fullSem);       // signal: one more space available
    return val;
}
```

## Why a Mutex Is Also Needed

The semaphores control **when** threads may enter the critical section, but if both `_emptySem` and `_fullSem` are > 0, multiple threads could pass the `sema_wait` simultaneously and corrupt `_head`, `_tail`, or `_size`. The mutex ensures that only one thread modifies the buffer state at a time.

## Example Traces

### Consumer Calls dequeue Before Producer Enqueues

| Step | Consumer | Producer | `_emptySem` | `_fullSem` |
|---|---|---|---|---|
| 1 | `sema_wait(&_emptySem)` — count=0, blocks | — | -1 (blocked) | MaxSize |
| 2 | — | `sema_wait(&_fullSem)` — proceeds | -1 | MaxSize-1 |
| 3 | — | inserts item; `sema_post(&_emptySem)` | 0 | MaxSize-1 |
| 4 | unblocks; acquires mutex; removes item; `sema_post(&_fullSem)` | — | 0 | MaxSize |

### Producer Fills Buffer Completely

| Step | Producer | Consumer | `_emptySem` | `_fullSem` |
|---|---|---|---|---|
| 1-N | enqueue N items | — | N | 0 |
| N+1 | `sema_wait(&_fullSem)` — count=0, blocks | — | N | -1 (blocked) |
| N+2 | — | `sema_wait(&_emptySem)` — proceeds; removes item; `sema_post(&_fullSem)` | N-1 | 0 |
| N+3 | unblocks; inserts item; `sema_post(&_emptySem)` | — | N | 0 |

## Notes

From the slides:
- `sema_wait` on `_fullSem` must happen **before** the mutex lock in `enqueue`.
- `sema_wait` on `_emptySem` must happen **before** the mutex lock in `dequeue`.
- Placing `sema_wait` inside the mutex would cause deadlock (producer holds mutex, waits for consumer to post, but consumer can't acquire mutex).

## Key Terms

| Term | Meaning |
|---|---|
| Bounded buffer | Fixed-size circular queue; producers add, consumers remove |
| `_emptySem` | Semaphore counting items in buffer; init=0; `sema_wait` in dequeue |
| `_fullSem` | Semaphore counting empty slots; init=MaxSize; `sema_wait` in enqueue |
| Producer | Thread calling `enqueue`; blocks when buffer is full |
| Consumer | Thread calling `dequeue`; blocks when buffer is empty |
