## The Vulnerable Program

From the lab:

```c
#include <stdio.h>
#include <string.h>
#include <stdlib.h>

int main(int argc, char *argv[]) {
    char buf[15];
    int set_me = 0;

    strcpy(buf, argv[1]);

    if (set_me == 0xdeadbeef) {
        printf("You win!\n");
        shell();
    } else {
        printf("set_me = %08x\n", set_me);
    }
}
```

`buf` and `set_me` are local variables stored on the **stack**. `set_me` is initialized to `0` and is located in memory **adjacent** to `buf`.

## The Vulnerability

`strcpy(buf, argv[1])` copies the command-line argument into `buf[15]` **without checking the length**.

If `argv[1]` is longer than 14 characters (plus the null terminator), `strcpy` writes past the end of `buf` and into adjacent stack memory — overwriting `set_me`.

## Stack Layout

On the stack, `set_me` sits just after `buf`:

```
[ buf[0] ... buf[14] ] [ set_me (4 bytes) ] ...
```

Writing 15 or more bytes into `buf` starts overwriting `set_me`.

## Little-Endian Encoding

From the lab:
> "x86 is a little-endian architecture: the least significant byte is stored at the lowest address."

The value `0xdeadbeef` must be written in **little-endian** byte order:

| Byte offset in set_me | Value |
|---|---|
| +0 (lowest address) | `0xef` |
| +1 | `0xbe` |
| +2 | `0xad` |
| +3 (highest address) | `0xde` |

As a byte string: `\xef\xbe\xad\xde`

## The Exploit Payload

From the lab:

```bash
./buffer `printf "AAAAAAAAAAAAAAA\xef\xbe\xad\xde"`
```

- 15 `A` characters fill `buf[0]` through `buf[14]`
- `\xef\xbe\xad\xde` overwrites `set_me` with `0xdeadbeef` (little-endian)
- The `if` condition becomes true and the program prints "You win!"

## Compile Flags Required

From the lab:

```bash
gcc -m32 -O0 -fno-stack-protector buffer.c -o buffer
```

| Flag | Purpose |
|---|---|
| `-m32` | Compile as 32-bit binary (simpler stack layout) |
| `-O0` | No optimization (preserves predictable variable layout) |
| `-fno-stack-protector` | Disables stack canary that would detect the overflow |

## Key Terms

| Term | Meaning |
|---|---|
| Buffer overflow | Writing past the end of a fixed-size buffer into adjacent memory |
| Stack layout | Local variables are placed on the stack in order; adjacent variables can be overwritten |
| Little-endian | Least significant byte stored at the lowest memory address (x86 architecture) |
| `strcpy` | Unsafe string copy; does not check destination buffer size |
| Stack canary | Security value placed after local buffers to detect overflow; disabled by `-fno-stack-protector` |
